# Calendly adapter

The Calendly adapter exposes scheduled events, event types, and invitees under `/calendly`, with writeback routes for creating scheduled event invitees, creating event types, updating event types, and canceling scheduled events.

Read-only mounts:
- `/calendly/scheduled-events/<eventId>.json` - Scheduled event records.
- `/calendly/scheduled-events/<eventId>/cancel.json` - Scheduled event cancellation writeback sidecar.
- `/calendly/event-types/<eventTypeId>.json` - Event type records.
- `/calendly/invitees/<inviteeId>.json` - Invitee records. Invitee writes are not supported by the public API.

Resources:

| Resource | Schema | Create example | ID pattern | What it does |
|---|---|---|---|---|
| `/calendly/scheduled-events/<id>.json` | `/calendly/scheduled-events/.schema.json` | `/calendly/scheduled-events/.create.example.json` | `^event_[A-Za-z0-9_-]+$` | Schedules an invitee for a Calendly event type at the requested start time. |
| `/calendly/event-types/<id>.json` | `/calendly/event-types/.schema.json` | `/calendly/event-types/.create.example.json` | `^type_[A-Za-z0-9_-]+$` | Creates a Calendly event type for the supplied owner. |

## ID Patterns
- `/calendly/scheduled-events/<id>.json`: `^event_[A-Za-z0-9_-]+$`. Filenames that do not match this pattern are treated as create drafts.
- `/calendly/event-types/<id>.json`: `^type_[A-Za-z0-9_-]+$`. Filenames that do not match this pattern are treated as create drafts.

## Write field contracts

### Create Calendly scheduled event invitee

Resource: `/calendly/scheduled-events/<id>.json`
Schema: `/calendly/scheduled-events/.schema.json`
Create example: `/calendly/scheduled-events/.create.example.json`
Required fields: `event_type`, `start_time`, `invitee`.
Optional fields: `location`, `timezone`, `questions_and_answers`, `tracking`.

Fields:

- `event_type` (required, string) - Calendly event type URI.
- `start_time` (required, string, date-time) - Requested event start time in ISO 8601 form.
- `invitee` (required, object) - Invitee details.
- `invitee.email` (required, string, email) - Invitee email address.
- `invitee.name` (required, string) - Invitee display name.
- `location` (optional, object) - Calendly location object.
- `timezone` (optional, string) - IANA timezone for the invitee.
- `questions_and_answers` (optional, array) - Invitee question answers.
- `tracking` (optional, object) - Calendly tracking fields.

### Create Calendly event type

Resource: `/calendly/event-types/<id>.json`
Schema: `/calendly/event-types/.schema.json`
Create example: `/calendly/event-types/.create.example.json`
Required fields: `name`, `owner`.
Optional fields: `duration`, `slug`, `active`, `color`, `description_plain`, `internal_note`.

Fields:

- `name` (required, string) - Event type name.
- `owner` (required, string) - Calendly user or organization owner URI.
- `duration` (optional, integer) - Event duration in minutes.
- `slug` (optional, string) - Event type slug.
- `active` (optional, boolean) - Whether the event type is active.
- `color` (optional, string) - Display color as a hex string.
- `description_plain` (optional, string) - Plain-text event type description.
- `internal_note` (optional, string) - Internal event type note.

Read the resource `.schema.json` first, then use the sibling `.create.example.json` as a minimal create document. The example intentionally omits read-only fields.
