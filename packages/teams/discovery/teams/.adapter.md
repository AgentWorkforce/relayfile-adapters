# Teams adapter

The Teams adapter exposes teams, channels, messages, replies, chats, tabs, members, and reactions under `/teams`, with writeback routes for creating channel messages, replies, and chat messages.

Read-only mounts:
- `/teams/<teamId>/metadata.json` - Team records.
- `/teams/<teamId>/channels/<channelId>/metadata.json` - Channel records.
- `/teams/<teamId>/channels/<channelId>/messages/<messageId>.json` - Channel message records.
- `/teams/chats/<chatId>/messages/<messageId>.json` - Chat message records.

Resources:

| Resource | Schema | Create example | ID pattern | What it does |
|---|---|---|---|---|
| `/teams/{teamId}/channels/{channelId}/messages/<id>.json` | `/teams/{teamId}/channels/{channelId}/messages/.schema.json` | `/teams/{teamId}/channels/{channelId}/messages/.create.example.json` | `^[A-Za-z0-9_.=!-]+$` | Posts a new Teams channel message. |
| `/teams/{teamId}/channels/{channelId}/messages/{messageId}/replies/<id>.json` | `/teams/{teamId}/channels/{channelId}/messages/{messageId}/replies/.schema.json` | `/teams/{teamId}/channels/{channelId}/messages/{messageId}/replies/.create.example.json` | `^[A-Za-z0-9_.=!-]+$` | Posts a reply to a Teams channel message. |
| `/teams/chats/{chatId}/messages/<id>.json` | `/teams/chats/{chatId}/messages/.schema.json` | `/teams/chats/{chatId}/messages/.create.example.json` | `^[A-Za-z0-9_.=!-]+$` | Posts a new Teams chat message. |

## Operations

| To... | Do... |
|---|---|
| Read | `cat <id>.json` |
| Edit | Write a partial JSON object to `<id>.json`. Only included mutable fields PATCH; fields marked `readOnly` in `.schema.json` are rejected. |
| Create | Write JSON to any non-canonical filename such as `create request.json`. The adapter creates the record at `<real-id>.json` and rewrites the draft as `{ "created": "<real-id>", "path": "<resource>/<real-id>.json", "url": "<provider-url>" }`. |
| Ignore | Editor scratch files named `partial.json`, `.tmp.json`, `.partial.json`, `*.tmp.json`, or `*.partial.json` are ignored and never treated as create drafts. |
| Delete | `rm <id>.json` for canonical ids. |

## ID Patterns
- `/teams/{teamId}/channels/{channelId}/messages/<id>.json`: `^[A-Za-z0-9_.=!-]+$`. Filenames that do not match this pattern are treated as create drafts.
- `/teams/{teamId}/channels/{channelId}/messages/{messageId}/replies/<id>.json`: `^[A-Za-z0-9_.=!-]+$`. Filenames that do not match this pattern are treated as create drafts.
- `/teams/chats/{chatId}/messages/<id>.json`: `^[A-Za-z0-9_.=!-]+$`. Filenames that do not match this pattern are treated as create drafts.

## Write field contracts

### Create Teams channel message

Resource: `/teams/{teamId}/channels/{channelId}/messages/<id>.json`
Schema: `/teams/{teamId}/channels/{channelId}/messages/.schema.json`
Create example: `/teams/{teamId}/channels/{channelId}/messages/.create.example.json`
Required fields: none at the top level.
Optional fields: `body`, `text`, `content`.
Validation: provide at least one of `body.content`, `text`, `content`.

Fields:

- `body` (optional, object) - Teams message body.
- `body.contentType` (optional, enum) - Message content type. The adapter sends html by default. Allowed values: `html`, `text`.
- `body.content` (required, string) - Message body content.
- `text` (optional, string) - Plain text or HTML message content. Used when `body.content` is omitted.
- `content` (optional, string) - Plain text or HTML message content. Used when `body.content` and `text` are omitted.
