# Box Storage Bridge Adapter

Box maps Box Webhooks v2 upload, copy, moved, deleted, and trashed triggers into Relayfile StorageBridgeEvent envelopes under /box/{accountId}/files/{id}.json. The adapter validates configuration, normalizes push notifications, publishes storage events, fetches content through injected provider clients, and resolves file-native writeback into provider-shaped operations.

## Read-only Mounts

Read-only synchronized content is mounted at `/box/{accountId}/...`. Deleted provider objects emit deletion events with null fetch content.

## Writable Resources

The resource `/box/files` ships `/box/files/.schema.json` and `/box/files/.create.example.json`.

The resource `/box/webhooks` ships `/box/webhooks/.schema.json` and `/box/webhooks/.create.example.json`.

## Operations

Create writes use any non-canonical filename in the resource directory. Canonical filenames patch mutable fields only. Removing a canonical file resolves to provider delete operations. Read-only fields such as `id`, `createdAt`, `updatedAt`, `url`, `_webhook`, and `_connection` are rejected before provider calls.

The object resource resolves to `/2.0/files/content`. The lifecycle resource resolves to `/2.0/webhooks`.

## Push and Auth Validation

Box signatures are validated with BOX-SIGNATURE-PRIMARY over the raw body when signingSecret is configured; x-relayfile-signature remains available for local CI.

## Nango template

Nango anchor: `box`; sync `box-files`; schemas derive from Box file and webhook v2 resources.

## Nango Fallback

Box declares `box-files` for scheduled fallback records using provider config key `box`.

## ID Patterns

`/box/files`: `^[0-9]+$`.

`/box/webhooks`: `^[0-9]+$`.
