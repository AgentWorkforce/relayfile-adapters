# ClickUp adapter

The ClickUp adapter exposes spaces, folders, lists, tasks, and comments under `/clickup`, with writeback routes for creating tasks, lists, folders, and task comments.

Read-only mounts:
- `/clickup/spaces/<spaceId>.json` - Space records.
- `/clickup/folders/<folderId>.json` - Folder records.
- `/clickup/lists/<listId>.json` - List records.
- `/clickup/tasks/<taskId>.json` - Task records.

Resources:

| Resource | Schema | Create example | ID pattern | What it does |
|---|---|---|---|---|
| `/clickup/tasks/{taskId}/comments/<id>.json` | `/clickup/tasks/{taskId}/comments/.schema.json` | `/clickup/tasks/{taskId}/comments/.create.example.json` | `^[A-Za-z0-9]+$` | Adds a comment to a ClickUp task. |
| `/clickup/lists/{listId}/tasks/<id>.json` | `/clickup/lists/{listId}/tasks/.schema.json` | `/clickup/lists/{listId}/tasks/.create.example.json` | `^[A-Za-z0-9]+$` | Creates a task in a ClickUp list. |
| `/clickup/folders/{folderId}/lists/<id>.json` | `/clickup/folders/{folderId}/lists/.schema.json` | `/clickup/folders/{folderId}/lists/.create.example.json` | `^[A-Za-z0-9]+$` | Creates a list inside a ClickUp folder. |
| `/clickup/spaces/{spaceId}/lists/<id>.json` | `/clickup/spaces/{spaceId}/lists/.schema.json` | `/clickup/spaces/{spaceId}/lists/.create.example.json` | `^[A-Za-z0-9]+$` | Creates a folderless list inside a ClickUp space. |
| `/clickup/spaces/{spaceId}/folders/<id>.json` | `/clickup/spaces/{spaceId}/folders/.schema.json` | `/clickup/spaces/{spaceId}/folders/.create.example.json` | `^[A-Za-z0-9]+$` | Creates a folder inside a ClickUp space. |

## Operations

| To... | Do... |
|---|---|
| Read | `cat <id>.json` |
| Edit | Write a partial JSON object to `<id>.json`. Only included mutable fields PATCH; fields marked `readOnly` in `.schema.json` are rejected. |
| Create | Write JSON to any non-canonical filename such as `draft-title.json`. The adapter creates the record at `<real-id>.json` and rewrites the draft as `{ "created": "<real-id>", "path": "<resource>/<real-id>.json", "url": "<provider-url>" }`. |
| Delete | `rm <id>.json` for canonical ids. |

## ID Patterns
- `/clickup/tasks/{taskId}/comments/<id>.json`: `^[A-Za-z0-9]+$`. Filenames that do not match this pattern are treated as create drafts.
- `/clickup/lists/{listId}/tasks/<id>.json`: `^[A-Za-z0-9]+$`. Filenames that do not match this pattern are treated as create drafts.
- `/clickup/folders/{folderId}/lists/<id>.json`: `^[A-Za-z0-9]+$`. Filenames that do not match this pattern are treated as create drafts.
- `/clickup/spaces/{spaceId}/lists/<id>.json`: `^[A-Za-z0-9]+$`. Filenames that do not match this pattern are treated as create drafts.
- `/clickup/spaces/{spaceId}/folders/<id>.json`: `^[A-Za-z0-9]+$`. Filenames that do not match this pattern are treated as create drafts.

## Write field contracts

### Create ClickUp task comment

Resource: `/clickup/tasks/{taskId}/comments/<id>.json`
Schema: `/clickup/tasks/{taskId}/comments/.schema.json`
Create example: `/clickup/tasks/{taskId}/comments/.create.example.json`
Required fields: `comment_text`.
Optional fields: none.

Fields:

- `comment_text` (required, string) - Comment body. A plain string body is also accepted by the resolver.

### Create ClickUp task

Resource: `/clickup/lists/{listId}/tasks/<id>.json`
Schema: `/clickup/lists/{listId}/tasks/.schema.json`
Create example: `/clickup/lists/{listId}/tasks/.create.example.json`
Required fields: `name`.
Optional fields: `description`, `markdown_description`, `assignees`, `tags`, `status`, `priority`, `due_date`, `due_date_time`, `start_date`, `start_date_time`, `time_estimate`, `points`, `notify_all`, `parent`, `links_to`, `check_required_custom_fields`, `custom_fields`.

Fields:

- `name` (required, string) - Task name.
- `description` (optional, string) - Plain-text task description.
- `markdown_description` (optional, string) - Markdown task description.
- `assignees` (optional, array) - Assignee user ids.
- `tags` (optional, array) - Task tags.
- `status` (optional, string) - Initial task status.
- `priority` (optional, integer) - ClickUp priority id.
- `due_date` (optional, integer) - Due date as Unix epoch milliseconds.
- `due_date_time` (optional, boolean) - Whether due_date includes a time.
- `start_date` (optional, integer) - Start date as Unix epoch milliseconds.
- `start_date_time` (optional, boolean) - Whether start_date includes a time.
- `time_estimate` (optional, integer) - Time estimate in milliseconds.
- `points` (optional, number) - Sprint points.
- `notify_all` (optional, boolean) - Whether to notify all watchers.
- `parent` (optional, string) - Parent task id.
- `links_to` (optional, string) - Task id this task links to.
- `check_required_custom_fields` (optional, boolean) - Whether ClickUp should validate required custom fields.
- `custom_fields` (optional, array) - Custom field assignments.
