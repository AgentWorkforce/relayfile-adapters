# ClickUp adapter

The ClickUp adapter exposes spaces, folders, lists, tasks, and comments under `/clickup`, with writeback routes for creating tasks, lists, folders, and task comments.

Read-only mounts:
- `/clickup/spaces/<spaceId>.json` - Space records.
- `/clickup/folders/<folderId>.json` - Folder records.
- `/clickup/lists/<listId>.json` - List records.
- `/clickup/tasks/<taskId>.json` - Task records.

Write endpoints:
| Path | Schema | What it does |
|---|---|---|
| `/clickup/tasks/{taskId}/comments/new.json` | `/clickup/tasks/{taskId}/comments/new.schema.json` | Adds a comment to a ClickUp task. |
| `/clickup/lists/{listId}/tasks/new.json` | `/clickup/lists/{listId}/tasks/new.schema.json` | Creates a task in a ClickUp list. |
| `/clickup/folders/{folderId}/lists/new.json` | `/clickup/folders/{folderId}/lists/new.schema.json` | Creates a list inside a ClickUp folder. |
| `/clickup/spaces/{spaceId}/lists/new.json` | `/clickup/spaces/{spaceId}/lists/new.schema.json` | Creates a folderless list inside a ClickUp space. |
| `/clickup/spaces/{spaceId}/folders/new.json` | `/clickup/spaces/{spaceId}/folders/new.schema.json` | Creates a folder inside a ClickUp space. |

## How to use (agents)
1. Read the relevant `*.schema.json` to learn the required shape.
2. Optional: read the matching `new.example.json` for a starter document.
3. Write your document to the `new.json` path.
4. Read back the most-recently-created sibling record written by the adapter.
