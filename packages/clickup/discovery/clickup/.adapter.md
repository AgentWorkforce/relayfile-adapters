# ClickUp adapter

The ClickUp adapter exposes spaces, folders, lists, tasks, and comments under `/clickup`, with writeback routes for creating tasks, lists, folders, and task comments.

Read-only mounts:
- `/clickup/spaces/<spaceId>.json` - Space records.
- `/clickup/folders/<folderId>.json` - Folder records.
- `/clickup/lists/<listId>.json` - List records.
- `/clickup/tasks/<taskId>.json` - Task records.

Write endpoints:

| Path | Schema | What it does |
|---|---|---|
| `/clickup/tasks/{taskId}/comments/new.json` | `/clickup/tasks/{taskId}/comments/new.schema.json` | Adds a comment to a ClickUp task. |
| `/clickup/lists/{listId}/tasks/new.json` | `/clickup/lists/{listId}/tasks/new.schema.json` | Creates a task in a ClickUp list. |
| `/clickup/folders/{folderId}/lists/new.json` | `/clickup/folders/{folderId}/lists/new.schema.json` | Creates a list inside a ClickUp folder. |
| `/clickup/spaces/{spaceId}/lists/new.json` | `/clickup/spaces/{spaceId}/lists/new.schema.json` | Creates a folderless list inside a ClickUp space. |
| `/clickup/spaces/{spaceId}/folders/new.json` | `/clickup/spaces/{spaceId}/folders/new.schema.json` | Creates a folder inside a ClickUp space. |

## Write field contracts

### Create ClickUp task comment

Path: `/clickup/tasks/{taskId}/comments/new.json`
Schema: `/clickup/tasks/{taskId}/comments/new.schema.json`
Example: `/clickup/tasks/{taskId}/comments/new.example.json`
Required fields: `comment_text`.
Optional fields: none.

Fields:

- `comment_text` (required, string) - Comment body. A plain string body is also accepted by the resolver.

### Create ClickUp task

Path: `/clickup/lists/{listId}/tasks/new.json`
Schema: `/clickup/lists/{listId}/tasks/new.schema.json`
Example: `/clickup/lists/{listId}/tasks/new.example.json`
Required fields: `name`.
Optional fields: `description`, `markdown_description`, `assignees`, `tags`, `status`, `priority`, `due_date`, `due_date_time`, `start_date`, `start_date_time`, `time_estimate`, `points`, `notify_all`, `parent`, `links_to`, `check_required_custom_fields`, `custom_fields`.

Fields:

- `name` (required, string) - Task name.
- `description` (optional, string) - Plain-text task description.
- `markdown_description` (optional, string) - Markdown task description.
- `assignees` (optional, array) - Assignee user ids.
- `tags` (optional, array) - Task tags.
- `status` (optional, string) - Initial task status.
- `priority` (optional, integer) - ClickUp priority id.
- `due_date` (optional, integer) - Due date as Unix epoch milliseconds.
- `due_date_time` (optional, boolean) - Whether due_date includes a time.
- `start_date` (optional, integer) - Start date as Unix epoch milliseconds.
- `start_date_time` (optional, boolean) - Whether start_date includes a time.
- `time_estimate` (optional, integer) - Time estimate in milliseconds.
- `points` (optional, number) - Sprint points.
- `notify_all` (optional, boolean) - Whether to notify all watchers.
- `parent` (optional, string) - Parent task id.
- `links_to` (optional, string) - Task id this task links to.
- `check_required_custom_fields` (optional, boolean) - Whether ClickUp should validate required custom fields.
- `custom_fields` (optional, array) - Custom field assignments.

### Create ClickUp folder list

Path: `/clickup/folders/{folderId}/lists/new.json`
Schema: `/clickup/folders/{folderId}/lists/new.schema.json`
Example: `/clickup/folders/{folderId}/lists/new.example.json`
Required fields: `name`.
Optional fields: `content`, `due_date`, `priority`, `assignee`, `status`.

Fields:

- `name` (required, string) - List name.
- `content` (optional, string) - List description.
- `due_date` (optional, integer) - Due date as Unix epoch milliseconds.
- `priority` (optional, integer) - ClickUp priority id.
- `assignee` (optional, string) - Assignee user id.
- `status` (optional, string) - List status.

## How to use (agents)
1. Read the relevant `*.schema.json` to learn the required shape.
2. Optional: read the matching `new.example.json` for a starter document.
3. Write your document to the `new.json` path.
4. Read back the most-recently-created sibling record written by the adapter.
