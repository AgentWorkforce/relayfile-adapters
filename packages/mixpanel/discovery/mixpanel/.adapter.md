# Mixpanel adapter

The Mixpanel adapter exposes events, profiles, and cohorts under `/mixpanel`, with writeback routes for tracking/importing events, setting/deleting profiles, and updating cohorts.

Read-only mounts:
- `/mixpanel/events/<eventId>.json` - Event records.
- `/mixpanel/events/<eventName>/import.json` - Historical event import writeback sidecar.
- `/mixpanel/profiles/<distinctId>.json` - Profile records.
- `/mixpanel/profiles/<distinctId>/delete.json` - Profile delete writeback sidecar.
- `/mixpanel/cohorts/<cohortId>.json` - Cohort records.

Resources:

| Resource | Schema | Create example | ID pattern | What it does |
|---|---|---|---|---|
| `/mixpanel/events/<id>.json` | `/mixpanel/events/.schema.json` | `/mixpanel/events/.create.example.json` | `^(?!(?:new\|create\|draft\|track)(?:[-_\s].*)?$)[A-Za-z0-9_.:-]+$` | Tracks a Mixpanel event. |
| `/mixpanel/profiles/<id>.json` | `/mixpanel/profiles/.schema.json` | `/mixpanel/profiles/.create.example.json` | `^(?!(?:new\|create\|draft\|track)(?:[-_\s].*)?$)[A-Za-z0-9_.:-]+$` | Sets profile properties for a Mixpanel distinct id. |
| `/mixpanel/cohorts/<id>.json` | `/mixpanel/cohorts/.schema.json` | `/mixpanel/cohorts/.create.example.json` | `^[A-Za-z0-9_.:-]+$` | Updates a Mixpanel cohort record. |

## Operations

| To... | Do... |
|---|---|
| Read | `cat <canonical-resource-path>` after listing the resource directory or following an alias when one is available. Use the resource table and ID patterns below to determine whether a resource uses a bare id, an adapter-specific slug/id filename, or an exact sidecar path such as `content.md`. |
| Edit | Write the resource update payload to the canonical resource path. For JSON resources, included mutable fields PATCH; fields marked `readOnly` in `.schema.json` are rejected. |
| Create | Write JSON to any non-canonical filename such as `create request.json`. The adapter creates the record at its canonical resource path and rewrites the draft as `{ "created": "<real-id>", "path": "<canonical-resource-path>", "url": "<provider-url>" }`. |
| Ignore | Editor scratch files named `partial.json`, `.tmp.json`, `.partial.json`, `*.tmp.json`, or `*.partial.json` are ignored and never treated as create drafts. |
| Delete | `rm <canonical-resource-path>` for canonical records. |

## ID Patterns
- `/mixpanel/events/<id>.json`: `^(?!(?:new|create|draft|track)(?:[-_\s].*)?$)[A-Za-z0-9_.:-]+$`. Filenames that do not match this pattern are treated as create drafts.
- `/mixpanel/profiles/<id>.json`: `^(?!(?:new|create|draft|track)(?:[-_\s].*)?$)[A-Za-z0-9_.:-]+$`. Filenames that do not match this pattern are treated as create drafts.
- `/mixpanel/cohorts/<id>.json`: `^[A-Za-z0-9_.:-]+$`. Filenames that do not match this pattern are treated as create drafts.

## Write field contracts

### Track Mixpanel event

Resource: `/mixpanel/events/<id>.json`
Schema: `/mixpanel/events/.schema.json`
Create example: `/mixpanel/events/.create.example.json`
Required fields: `event`.
Optional fields: `name`, `distinct_id`, `time`, `properties`.

Fields:

- `event` (required, string) - Event name. The resolver also accepts `name`.
- `name` (optional, string) - Event name alias.
- `distinct_id` (optional, string) - Profile distinct id. Copied into `properties.distinct_id`.
- `time` (optional, number) - Event time as a Unix timestamp.
- `properties` (optional, object) - Mixpanel event properties.

### Set Mixpanel profile

Resource: `/mixpanel/profiles/<id>.json`
Schema: `/mixpanel/profiles/.schema.json`
Create example: `/mixpanel/profiles/.create.example.json`
Required fields: `distinct_id`.
Optional fields: `$distinct_id`, `properties`, `$set`, `$properties`.

Fields:

- `distinct_id` (required, string) - Profile distinct id. `$distinct_id` is also accepted by the resolver.
- `$distinct_id` (optional, string) - Profile distinct id.
- `properties` (optional, object) - Profile properties.
- `$set` (optional, object) - Profile properties to set.
- `$properties` (optional, object) - Profile properties to set.

### Update Mixpanel cohort

Resource: `/mixpanel/cohorts/<id>.json`
Schema: `/mixpanel/cohorts/.schema.json`
Create example: `/mixpanel/cohorts/.create.example.json`
Required fields: `name`.
Optional fields: `description`.

Fields:

- `name` (required, string) - Cohort name.
- `description` (optional, string) - Cohort description.

## Create Examples
Read the resource `.schema.json` first, then use the sibling `.create.example.json` as a minimal create document. The example intentionally omits read-only fields.
