# Asana adapter

The Asana adapter exposes task, project, section, and workspace records under `/asana`, with writeback routes for creating tasks, projects, and sections plus updating existing records.

Read-only mounts:
- `/asana/tasks/<taskId>.json` - Task records.
- `/asana/projects/<projectId>.json` - Project records.
- `/asana/projects/<projectId>/sections/<sectionId>.json` - Project-scoped section records.
- `/asana/workspaces/<workspaceId>.json` - Workspace records.

Resources:

| Resource | Schema | Create example | ID pattern | What it does |
|---|---|---|---|---|
| `/asana/tasks/<id>.json` | `/asana/tasks/.schema.json` | `/asana/tasks/.create.example.json` | `^\d+$` | Creates an Asana task. |
| `/asana/projects/<id>.json` | `/asana/projects/.schema.json` | `/asana/projects/.create.example.json` | `^\d+$` | Creates an Asana project. |
| `/asana/sections/<id>.json` | `/asana/sections/.schema.json` | `/asana/sections/.create.example.json` | `^\d+$` | Creates an Asana section when the project gid is supplied in the document. |
| `/asana/projects/{projectId}/sections/<id>.json` | `/asana/projects/{projectId}/sections/.schema.json` | `/asana/projects/{projectId}/sections/.create.example.json` | `^\d+$` | Creates a section inside the project named by the path. |

## Operations

| To... | Do... |
|---|---|
| Read | `cat <id>.json` |
| Edit | Write a partial JSON object to `<id>.json`. Only included mutable fields PATCH; fields marked `readOnly` in `.schema.json` are rejected. |
| Create | Write JSON to any non-canonical filename such as `draft-title.json`. The adapter creates the record at `<real-id>.json` and rewrites the draft as `{ "created": "<real-id>", "path": "<resource>/<real-id>.json", "url": "<provider-url>" }`. |
| Delete | `rm <id>.json` for canonical ids. |

## ID Patterns
- `/asana/tasks/<id>.json`: `^\d+$`. Filenames that do not match this pattern are treated as create drafts.
- `/asana/projects/<id>.json`: `^\d+$`. Filenames that do not match this pattern are treated as create drafts.
- `/asana/sections/<id>.json`: `^\d+$`. Filenames that do not match this pattern are treated as create drafts.
- `/asana/projects/{projectId}/sections/<id>.json`: `^\d+$`. Filenames that do not match this pattern are treated as create drafts.

## Write field contracts

### Create Asana task

Resource: `/asana/tasks/<id>.json`
Schema: `/asana/tasks/.schema.json`
Create example: `/asana/tasks/.create.example.json`
Required fields: `name`.
Optional fields: `workspace`, `assignee`, `assignee_status`, `notes`, `html_notes`, `due_on`, `due_at`, `start_on`, `start_at`, `completed`, `projects`, `followers`, `tags`, `custom_fields`, `parent`.

Fields:

- `name` (required, string) - Task name.
- `workspace` (optional, string) - Workspace gid for the new task.
- `assignee` (optional, string) - User gid or `me` to assign the task.
- `assignee_status` (optional, string) - Assignee status such as inbox, later, upcoming, or today.
- `notes` (optional, string) - Plain-text task notes.
- `html_notes` (optional, string) - HTML task notes.
- `due_on` (optional, string, date) - Due date in YYYY-MM-DD form.
- `due_at` (optional, string, date-time) - Due timestamp in ISO 8601 form.
- `start_on` (optional, string, date) - Start date in YYYY-MM-DD form.
- `start_at` (optional, string, date-time) - Start timestamp in ISO 8601 form.
- `completed` (optional, boolean) - Whether the task starts completed.
- `projects` (optional, array) - Project gids to add the task to.
- `followers` (optional, array) - Follower user gids.
- `tags` (optional, array) - Tag gids to attach.
- `custom_fields` (optional, object) - Asana custom field gid to value map.
- `parent` (optional, string) - Parent task gid for subtasks.

### Create Asana project

Resource: `/asana/projects/<id>.json`
Schema: `/asana/projects/.schema.json`
Create example: `/asana/projects/.create.example.json`
Required fields: `name`.
Optional fields: `workspace`, `team`, `notes`, `color`, `default_view`, `due_on`, `start_on`, `public`, `archived`, `custom_fields`.

Fields:

- `name` (required, string) - Project name.
- `workspace` (optional, string) - Workspace gid for the project.
- `team` (optional, string) - Team gid for the project.
- `notes` (optional, string) - Project notes.
- `color` (optional, string) - Asana project color name.
- `default_view` (optional, string) - Initial project view.
- `due_on` (optional, string, date) - Project due date in YYYY-MM-DD form.
- `start_on` (optional, string, date) - Project start date in YYYY-MM-DD form.
- `public` (optional, boolean) - Whether the project is public to the workspace.
- `archived` (optional, boolean) - Whether the project starts archived.
- `custom_fields` (optional, object) - Asana custom field gid to value map.

### Create Asana section

Resource: `/asana/sections/<id>.json`
Schema: `/asana/sections/.schema.json`
Create example: `/asana/sections/.create.example.json`
Required fields: `name`, `project`.
Optional fields: none.

Fields:

- `name` (required, string) - Section name.
- `project` (required, string) - Project gid that will contain the section.

### Create Asana project section

Resource: `/asana/projects/{projectId}/sections/<id>.json`
Schema: `/asana/projects/{projectId}/sections/.schema.json`
Create example: `/asana/projects/{projectId}/sections/.create.example.json`
Required fields: `name`.
Optional fields: none.

Fields:

- `name` (required, string) - Section name.

## Create Examples
Read the resource `.schema.json` first, then use the sibling `.create.example.json` as a minimal create document. The example intentionally omits read-only fields.
