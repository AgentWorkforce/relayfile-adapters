# Azure Blob Storage adapter

The Azure Blob Storage adapter exposes blobs and Event Grid subscriptions with file-native writeback discovery.

Read-only mounts:
- `/azure/<account>/<container>/<blob>` - Azure blob content.

Resources:

| Resource | Schema | Create example | ID pattern | What it does |
|---|---|---|---|---|
| `/azure-blob/blobs/<id>.json` | `/azure-blob/blobs/.schema.json` | `/azure-blob/blobs/.create.example.json` | `^[A-Za-z0-9_.:-]+$` | Uploads a blob. |
| `/azure-blob/event-subscriptions/<id>.json` | `/azure-blob/event-subscriptions/.schema.json` | `/azure-blob/event-subscriptions/.create.example.json` | `^[A-Za-z0-9_.:-]+$` | Creates a storage Event Grid subscription. |

## Operations

| To... | Do... |
|---|---|
| Read | `cat <canonical-file>.json` after listing the resource directory or following an alias when one is available. Canonical files may be bare ids or slug-prefixed names such as `<slug>__<id>.json`. |
| Edit | Write a partial JSON object to the canonical file. Only included mutable fields PATCH; fields marked `readOnly` in `.schema.json` are rejected. |
| Create | Write JSON to any non-canonical filename such as `create request.json`. The adapter creates the record at its canonical filename and rewrites the draft as `{ "created": "<real-id>", "path": "<resource>/<canonical-file>.json", "url": "<provider-url>" }`. |
| Ignore | Editor scratch files named `partial.json`, `.tmp.json`, `.partial.json`, `*.tmp.json`, or `*.partial.json` are ignored and never treated as create drafts. |
| Delete | `rm <canonical-file>.json` for canonical records. |

## ID Patterns
- `/azure-blob/blobs/<id>.json`: `^[A-Za-z0-9_.:-]+$`. Filenames that do not match this pattern are treated as create drafts.
- `/azure-blob/event-subscriptions/<id>.json`: `^[A-Za-z0-9_.:-]+$`. Filenames that do not match this pattern are treated as create drafts.

## Write field contracts

### Create Azure blob

Resource: `/azure-blob/blobs/<id>.json`
Schema: `/azure-blob/blobs/.schema.json`
Create example: `/azure-blob/blobs/.create.example.json`
Required fields: `account`, `container`, `name`.
Optional fields: `contentBase64`.

Fields:

- `account` (required, string) - Azure account.
- `container` (required, string) - Container.
- `name` (required, string) - Blob name.
- `contentBase64` (optional, string) - Base64 content.

### Create Azure Event Grid subscription

Resource: `/azure-blob/event-subscriptions/<id>.json`
Schema: `/azure-blob/event-subscriptions/.schema.json`
Create example: `/azure-blob/event-subscriptions/.create.example.json`
Required fields: `endpointUrl`.
Optional fields: `includedEventTypes`.

Fields:

- `endpointUrl` (required, string) - Webhook endpoint URL.
- `includedEventTypes` (optional, array) - Included event types.

## Create Examples
Read the resource `.schema.json` first, then use the sibling `.create.example.json` as a minimal create document. The example intentionally omits read-only fields.
