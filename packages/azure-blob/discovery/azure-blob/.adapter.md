# Azure Blob Storage Storage Bridge Adapter

Azure Blob Storage maps Azure Event Grid BlobCreated and BlobDeleted events into Relayfile StorageBridgeEvent envelopes under /azure/{account}/{container}/{name}. The adapter validates configuration, normalizes push notifications, publishes storage events, fetches content through injected provider clients, and resolves file-native writeback into provider-shaped operations.

## Read-only Mounts

Read-only synchronized content is mounted at `/azure/{account}/{container}/{name}`. Deleted provider objects emit deletion events with null fetch content.

## Writable Resources

The resource `/azure/{account}/{container}/blobs` ships `/azure-blob/azure/{account}/{container}/blobs/.schema.json` and `/azure-blob/azure/{account}/{container}/blobs/.create.example.json`.

The resource `/azure/event-subscriptions` ships `/azure-blob/azure/event-subscriptions/.schema.json` and `/azure-blob/azure/event-subscriptions/.create.example.json`.

The static discovery resource `/azure-blob/blobs/.schema.json` ships `/azure-blob/blobs/.create.example.json`.

The static discovery resource `/azure-blob/event-subscriptions/.schema.json` ships `/azure-blob/event-subscriptions/.create.example.json`.

## Operations

Create writes use any non-canonical filename in the resource directory. Canonical filenames patch mutable fields only. Removing a canonical file resolves to provider delete operations. Read-only fields such as `id`, `createdAt`, `updatedAt`, `url`, `_webhook`, and `_connection` are rejected before provider calls.

The object resource resolves to `/{container}/{name}`. The lifecycle resource resolves to `/eventSubscriptions/{id}`.

## Push and Auth Validation

Event Grid subscription validation events are returned by getWebhookChallenge; production deployments should pair this with shared access signature or Entra edge validation. CI can use x-relayfile-signature HMAC.

## Nango template

No Nango anchor; spec source: Azure Blob Storage REST API and Event Grid Storage event schema.

## Nango Fallback

Azure Blob Storage is not in the Nango scheduled fallback catalog for this wave; operators must rely on the real-time push path.

## ID Patterns

`/azure/{account}/{container}/blobs`: `^.+$`.

`/azure/event-subscriptions`: `^[A-Za-z0-9._-]+$`.
