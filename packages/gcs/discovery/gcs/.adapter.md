# Google Cloud Storage adapter

The GCS adapter exposes objects and Pub/Sub notification configs with file-native writeback discovery.

Read-only mounts:
- `/gcs/<bucket>/<object>` - GCS object content.

Resources:

| Resource | Schema | Create example | ID pattern | What it does |
|---|---|---|---|---|
| `/gcs/objects/<id>.json` | `/gcs/objects/.schema.json` | `/gcs/objects/.create.example.json` | `^[A-Za-z0-9_.:-]+$` | Uploads a GCS object. |
| `/gcs/notifications/<id>.json` | `/gcs/notifications/.schema.json` | `/gcs/notifications/.create.example.json` | `^[A-Za-z0-9_.:-]+$` | Creates a bucket notification. |

## Operations

| To... | Do... |
|---|---|
| Read | `cat <id>.json` |
| Edit | Write a partial JSON object to `<id>.json`. Only included mutable fields PATCH; fields marked `readOnly` in `.schema.json` are rejected. |
| Create | Write JSON to any non-canonical filename such as `create request.json`. The adapter creates the record at `<real-id>.json` and rewrites the draft as `{ "created": "<real-id>", "path": "<resource>/<real-id>.json", "url": "<provider-url>" }`. |
| Delete | `rm <id>.json` for canonical ids. |

## ID Patterns
- `/gcs/objects/<id>.json`: `^[A-Za-z0-9_.:-]+$`. Filenames that do not match this pattern are treated as create drafts.
- `/gcs/notifications/<id>.json`: `^[A-Za-z0-9_.:-]+$`. Filenames that do not match this pattern are treated as create drafts.

## Write field contracts

### Create GCS object

Resource: `/gcs/objects/<id>.json`
Schema: `/gcs/objects/.schema.json`
Create example: `/gcs/objects/.create.example.json`
Required fields: `bucket`, `name`.
Optional fields: `contentBase64`.

Fields:

- `bucket` (required, string) - Bucket.
- `name` (required, string) - Object name.
- `contentBase64` (optional, string) - Base64 content.

### Create GCS notification

Resource: `/gcs/notifications/<id>.json`
Schema: `/gcs/notifications/.schema.json`
Create example: `/gcs/notifications/.create.example.json`
Required fields: `bucket`, `topic`.
Optional fields: none.

Fields:

- `bucket` (required, string) - Bucket.
- `topic` (required, string) - Pub/Sub topic.

## Create Examples
Read the resource `.schema.json` first, then use the sibling `.create.example.json` as a minimal create document. The example intentionally omits read-only fields.
