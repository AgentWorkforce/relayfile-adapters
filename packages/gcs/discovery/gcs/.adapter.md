# Google Cloud Storage Storage Bridge Adapter

Google Cloud Storage maps Cloud Pub/Sub OBJECT_FINALIZE, OBJECT_DELETE, and OBJECT_METADATA_UPDATE notifications into Relayfile StorageBridgeEvent envelopes under /gcs/{bucket}/{name}. The adapter validates configuration, normalizes push notifications, publishes storage events, fetches content through injected provider clients, and resolves file-native writeback into provider-shaped operations.

## Read-only Mounts

Read-only synchronized content is mounted at `/gcs/{bucket}/{name}`. Deleted provider objects emit deletion events with null fetch content.

## Writable Resources

The resource `/gcs/{bucket}/objects` ships `/gcs/{bucket}/objects/.schema.json` and `/gcs/{bucket}/objects/.create.example.json`.

The resource `/gcs/notifications/{bucketId}` ships `/gcs/notifications/{bucketId}/.schema.json` and `/gcs/notifications/{bucketId}/.create.example.json`.

The static discovery resource `/gcs/objects/.schema.json` ships `/gcs/objects/.create.example.json`.

The static discovery resource `/gcs/notifications/.schema.json` ships `/gcs/notifications/.create.example.json`.

## Operations

Create writes use any non-canonical filename in the resource directory. Canonical filenames patch mutable fields only. Removing a canonical file resolves to provider delete operations. Read-only fields such as `id`, `createdAt`, `updatedAt`, `url`, `_webhook`, and `_connection` are rejected before provider calls.

The object resource resolves to `/storage/v1/b/{bucket}/o/{name}`. The lifecycle resource resolves to `/storage/v1/b/{bucket}/notificationConfigs`.

## Push and Auth Validation

Cloud Pub/Sub push requests can be validated with an OIDC bearer token by the hosting layer; this package also supports a deterministic x-relayfile-signature HMAC for local and CI receivers.

## Nango template

No Nango anchor; spec source: Google Cloud Storage JSON API objects and Pub/Sub storage notification payloads.

## Nango Fallback

Google Cloud Storage is not in the Nango scheduled fallback catalog for this wave; operators must rely on the real-time push path.

## ID Patterns

`/gcs/{bucket}/objects`: `^.+$`.

`/gcs/notifications/{bucketId}`: `^[A-Za-z0-9._-]+$`.
