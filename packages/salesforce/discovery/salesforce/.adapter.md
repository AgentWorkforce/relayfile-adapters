# Salesforce adapter

The Salesforce adapter exposes Account, Contact, Opportunity, Lead, and Case sObjects under `/salesforce`, with writeback routes for creating and updating those records.

Read-only mounts:
- `/salesforce/accounts/<accountId>.json` - Account records.
- `/salesforce/contacts/<contactId>.json` - Contact records.
- `/salesforce/opportunities/<opportunityId>.json` - Opportunity records.
- `/salesforce/leads/<leadId>.json` - Lead records.
- `/salesforce/cases/<caseId>.json` - Case records.

Write endpoints:

| Path | Schema | What it does |
|---|---|---|
| `/salesforce/accounts/new.json` | `/salesforce/accounts/new.schema.json` | Creates a Salesforce Account. |
| `/salesforce/contacts/new.json` | `/salesforce/contacts/new.schema.json` | Creates a Salesforce Contact. |
| `/salesforce/opportunities/new.json` | `/salesforce/opportunities/new.schema.json` | Creates a Salesforce Opportunity. |
| `/salesforce/leads/new.json` | `/salesforce/leads/new.schema.json` | Creates a Salesforce Lead. |
| `/salesforce/cases/new.json` | `/salesforce/cases/new.schema.json` | Creates a Salesforce Case. |

## Write field contracts

### Create Salesforce account

Path: `/salesforce/accounts/new.json`
Schema: `/salesforce/accounts/new.schema.json`
Example: `/salesforce/accounts/new.example.json`
Required fields: `Name`.
Optional fields: `Website`, `Phone`, `Industry`, `Type`, `BillingStreet`, `BillingCity`, `BillingState`, `BillingPostalCode`, `BillingCountry`, `OwnerId`.

Fields:

- `Name` (required, string) - Account name.
- `Website` (optional, string, uri) - Account website URL.
- `Phone` (optional, string) - Account phone number.
- `Industry` (optional, string) - Account industry.
- `Type` (optional, string) - Account type.
- `BillingStreet` (optional, string) - Billing street address.
- `BillingCity` (optional, string) - Billing city.
- `BillingState` (optional, string) - Billing state or province.
- `BillingPostalCode` (optional, string) - Billing postal code.
- `BillingCountry` (optional, string) - Billing country.
- `OwnerId` (optional, string) - Owner user id.
- additional properties (optional, one of string, number, integer, boolean, null, object, array) - Writable Salesforce sObject field value. Use Salesforce object metadata to discover required fields for the object type.

## How to use (agents)
1. Read the relevant `*.schema.json` to learn the required shape.
2. Optional: read the matching `new.example.json` for a starter document.
3. Write your document to the `new.json` path.
4. Read back the most-recently-created sibling record written by the adapter.
