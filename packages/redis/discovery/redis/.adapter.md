# Redis Storage Bridge Adapter

Redis maps Redis keyspace notifications through __keyspace@{db}__:* patterns into Relayfile StorageBridgeEvent envelopes under /redis/{db}/{key}. The adapter validates configuration, normalizes push notifications, publishes storage events, fetches content through injected provider clients, and resolves file-native writeback into provider-shaped operations.

## Read-only Mounts

Read-only synchronized content is mounted at `/redis/{db}/{key}[.json]`. Deleted provider objects emit deletion events with null fetch content.

## Writable Resources

The resource `/redis/{db}` ships `/redis/{db}/.schema.json` and `/redis/{db}/.create.example.json`.

The resource `/redis/listeners` ships `/redis/listeners/.schema.json` and `/redis/listeners/.create.example.json`.

The static discovery resource `/redis/keys/.schema.json` ships `/redis/keys/.create.example.json`.

## Operations

Create writes use any non-canonical filename in the resource directory. Canonical filenames patch mutable fields only. Removing a canonical file resolves to provider delete operations. Read-only fields such as `id`, `createdAt`, `updatedAt`, `url`, `_webhook`, and `_connection` are rejected before provider calls.

The object resource resolves to `SET/HSET/ZADD/DEL`. The lifecycle resource resolves to `PSUBSCRIBE __keyspace@{db}__:*`.

## Push and Auth Validation

Keyspace events are trusted through the configured Redis connection. Local webhook fixtures can use x-relayfile-signature HMAC.

## Nango template

No Nango anchor; spec source: Redis keyspace notifications and Redis key write commands.

## Nango Fallback

Redis is not in the Nango scheduled fallback catalog for this wave; operators must rely on the real-time push path.

## ID Patterns

`/redis/{db}`: `^.+$`.

`/redis/listeners`: `^[0-9]+$`.
