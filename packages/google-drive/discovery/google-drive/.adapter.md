# Google Drive Storage Bridge Adapter

Google Drive maps Drive Watch API plus Changes API into Relayfile StorageBridgeEvent envelopes under /google-drive/{accountId}/path/to/file.ext. The adapter validates configuration, normalizes push notifications, publishes storage events, fetches content through injected provider clients, and resolves file-native writeback into provider-shaped operations.

## Read-only Mounts

Read-only synchronized content is mounted at `/google-drive/{accountId}/...`. Deleted provider objects emit deletion events with null fetch content.

## Writable Resources

The resource `/google-drive/files` ships `/google-drive/files/.schema.json` and `/google-drive/files/.create.example.json`.

The resource `/google-drive/channels` ships `/google-drive/channels/.schema.json` and `/google-drive/channels/.create.example.json`.

## Operations

Create writes use any non-canonical filename in the resource directory. Canonical filenames patch mutable fields only. Removing a canonical file resolves to provider delete operations. Read-only fields such as `id`, `createdAt`, `updatedAt`, `url`, `_webhook`, and `_connection` are rejected before provider calls.

The object resource resolves to `/drive/v3/files`. The lifecycle resource resolves to `/drive/v3/files/{resourceId}/watch`.

## Push and Auth Validation

Drive channel callbacks validate the configured channel token when Google sends x-goog-channel-token; local tests can use the x-relayfile-signature HMAC header.

## Nango template

Nango anchor: `google-drive`; sync `google-drive-files`; scopes `https://www.googleapis.com/auth/drive.readonly` and `https://www.googleapis.com/auth/drive`. The adapter intentionally keeps writeback-capable file schemas aligned with Drive v3 file and channel resources.

## Nango Fallback

Google Drive declares `google-drive-files` for scheduled fallback records using provider config key `google-drive`.

## ID Patterns

`/google-drive/files`: `^[A-Za-z0-9_-]+$`.

`/google-drive/channels`: `^[A-Za-z0-9_-]+$`.
