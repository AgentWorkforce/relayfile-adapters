# Google Calendar Adapter

## Overview

The Google Calendar adapter maps calendars and events into Relayfile paths under `/google-calendar`. Google push notifications are treated as change signals only; the adapter refreshes event data with `events.list` and stores event JSON in Relayfile.

## Read-Only Mount Summary

The adapter syncs Google Calendar event resources from one calendar at a time, defaulting to `primary`. Calendar metadata can be materialized at `/google-calendar/calendars/<calendarId>.json`, and event files are materialized under `/google-calendar/calendars/<calendarId>/events/<eventId>.json`.

## Resource Table

| Resource | Path | Schema | Create Example | ID Pattern |
| --- | --- | --- | --- | --- |
| events | `/google-calendar/calendars/{calendarId}/events/{eventId}.json` | `discovery/events.schema.json` | `discovery/events.create.example.json` | `^[a-v0-9]{5,1024}$` |

## Operation Table

| Operation | Method | Provider Endpoint | Relayfile Behavior |
| --- | --- | --- | --- |
| Create event | `POST` | `/calendar/v3/calendars/{calendarId}/events` | Write a valid event create document to a non-canonical filename in the target events directory. |
| Update event | `PATCH` | `/calendar/v3/calendars/{calendarId}/events/{eventId}` | Edit mutable fields in a canonical event JSON file. |
| Delete event | `DELETE` | `/calendar/v3/calendars/{calendarId}/events/{eventId}` | Remove a canonical event JSON file. |

## ID Pattern

Google Calendar event IDs use base32hex-style characters. The adapter treats `^[a-v0-9]{5,1024}$` as canonical for event update and delete paths. Non-canonical event filenames are reserved for create documents.
