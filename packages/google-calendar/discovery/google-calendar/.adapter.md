# Google Calendar adapter

The Google Calendar adapter exposes calendar and event records under `/google-calendar`, with writeback routes for creating, patching, and deleting events.

Read-only mounts:
- `/google-calendar/calendars/<calendarId>.json` - Calendar metadata.
- `/google-calendar/calendars/<calendarId>/events/<eventId>.json` - Calendar event records.

Resources:

| Resource | Schema | Create example | ID pattern | What it does |
|---|---|---|---|---|
| `/google-calendar/calendars/{calendarId}/events/<id>.json` | `/google-calendar/calendars/{calendarId}/events/.schema.json` | `/google-calendar/calendars/{calendarId}/events/.create.example.json` | `^[a-v0-9]{5,1024}$` | Creates a calendar event in the calendar named by the path. |

## Operations

| To... | Do... |
|---|---|
| Read | `cat <id>.json` |
| Edit | Write a partial JSON object to `<id>.json`. Only included mutable fields PATCH; fields marked `readOnly` in `.schema.json` are rejected. |
| Create | Write JSON to any non-canonical filename such as `draft-title.json`. The adapter creates the record at `<real-id>.json` and rewrites the draft as `{ "created": "<real-id>", "path": "<resource>/<real-id>.json", "url": "<provider-url>" }`. |
| Delete | `rm <id>.json` for canonical ids. |

## ID Patterns
- `/google-calendar/calendars/{calendarId}/events/<id>.json`: `^[a-v0-9]{5,1024}$`. Filenames that do not match this pattern are treated as create drafts.

## Write field contracts

### Create Google Calendar event

Resource: `/google-calendar/calendars/{calendarId}/events/<id>.json`
Schema: `/google-calendar/calendars/{calendarId}/events/.schema.json`
Create example: `/google-calendar/calendars/{calendarId}/events/.create.example.json`
Required fields: `start`, `end`.
Optional fields: `summary`, `description`, `location`, `attendees`, `status`, `recurrence`, `reminders`, `conferenceData`, `transparency`, `visibility`, `colorId`.
Validation: provide at least one of `summary`, `description`, `location`, `attendees`, `recurrence`, `conferenceData`.

Fields:

- `summary` (optional, string) - Event title.
- `description` (optional, string) - Event description. Google Calendar accepts HTML.
- `location` (optional, string) - Free-form event location.
- `start` (required, object) - Google Calendar event time.
- `start.date` (optional, string, date) - All-day event date in YYYY-MM-DD form.
- `start.dateTime` (optional, string, date-time) - Timed event timestamp in RFC 3339 form.
- `start.timeZone` (optional, string) - IANA time zone name.
- `end` (required, object) - Google Calendar event time.
- `end.date` (optional, string, date) - All-day event date in YYYY-MM-DD form.
- `end.dateTime` (optional, string, date-time) - Timed event timestamp in RFC 3339 form.
- `end.timeZone` (optional, string) - IANA time zone name.
- `attendees` (optional, array) - Event attendees.
- `attendees[].email` (optional, string, email) - Attendee email address.
- `attendees[].displayName` (optional, string) - Attendee display name.
- `attendees[].optional` (optional, boolean) - Whether attendance is optional.
- `attendees[].resource` (optional, boolean) - Whether the attendee is a room or resource.
- `attendees[].responseStatus` (optional, enum) - Attendee response status. Allowed values: `needsAction`, `declined`, `tentative`, `accepted`.
- `status` (optional, enum) - Event status. Allowed values: `confirmed`, `tentative`, `cancelled`.
- `recurrence` (optional, array) - Recurrence lines.
- `reminders` (optional, object) - Reminder settings.
- `reminders.useDefault` (optional, boolean) - Whether to use calendar default reminders.
- `reminders.overrides` (optional, array) - Reminder overrides.
- `reminders.overrides[].method` (optional, enum) - Reminder delivery method. Allowed values: `email`, `popup`.
- `reminders.overrides[].minutes` (optional, integer) - Minutes before the event start.
- `conferenceData` (optional, object) - Conference data such as a Google Meet create request.
- `transparency` (optional, enum) - Whether the event blocks calendar availability. Allowed values: `opaque`, `transparent`.
- `visibility` (optional, enum) - Event visibility. Allowed values: `default`, `public`, `private`, `confidential`.
- `colorId` (optional, string) - Google Calendar color id.

## Create Examples
Read the resource `.schema.json` first, then use the sibling `.create.example.json` as a minimal create document. The example intentionally omits read-only fields.
