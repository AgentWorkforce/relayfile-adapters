# GitHub adapter

The GitHub adapter exposes repository pull requests, issues, reviews, comments, commits, files, and checks under `/github`, with writeback support for creating and updating issues, creating and updating issue comments, and submitting pull request reviews.

Read-only mounts:
- `/github/repos/<owner>/<repo>/pulls/<pullNumber>/metadata.json` - Pull request metadata.
- `/github/repos/<owner>/<repo>/pulls/<pullNumber>/files/<path>` - Pull request file records.
- `/github/repos/<owner>/<repo>/issues/<issueNumber>/metadata.json` - Issue metadata.
- `/github/repos/<owner>/<repo>/commits/<sha>/metadata.json` - Commit metadata.

Resources:

| Resource | Schema | Create example | ID pattern | What it does |
|---|---|---|---|---|
| `/github/repos/{owner}/{repo}/issues/<id>.json` | `/github/repos/{owner}/{repo}/issues/.schema.json` | `/github/repos/{owner}/{repo}/issues/.create.example.json` | `^[1-9]\d*$` | Creates or updates a GitHub issue. |
| `/github/repos/{owner}/{repo}/issues/{issueNumber}/comments/<id>.json` | `/github/repos/{owner}/{repo}/issues/{issueNumber}/comments/.schema.json` | `/github/repos/{owner}/{repo}/issues/{issueNumber}/comments/.create.example.json` | `^\d+$` | Creates or updates a GitHub issue comment. |
| `/github/repos/{owner}/{repo}/pulls/{pullNumber}/reviews/<id>.json` | `/github/repos/{owner}/{repo}/pulls/{pullNumber}/reviews/.schema.json` | `/github/repos/{owner}/{repo}/pulls/{pullNumber}/reviews/.create.example.json` | `^\d+$` | Submits a pull request review with optional inline comments. |

## Operations

| To... | Do... |
|---|---|
| Read | `cat <id>.json` |
| Edit | Write a partial JSON object to `<id>.json`. Only included mutable fields PATCH; fields marked `readOnly` in `.schema.json` are rejected. |
| Create | Write JSON to any non-canonical filename such as `create request.json`. The adapter creates the record at `<real-id>.json` and rewrites the draft as `{ "created": "<real-id>", "path": "<resource>/<real-id>.json", "url": "<provider-url>" }`. |
| Delete | `rm <id>.json` for canonical ids. |

## ID Patterns
- `/github/repos/{owner}/{repo}/issues/<id>.json`: `^[1-9]\d*$`. Filenames that do not match this pattern are treated as create drafts.
- `/github/repos/{owner}/{repo}/issues/{issueNumber}/comments/<id>.json`: `^\d+$`. Filenames that do not match this pattern are treated as create drafts.
- `/github/repos/{owner}/{repo}/pulls/{pullNumber}/reviews/<id>.json`: `^\d+$`. Filenames that do not match this pattern are treated as create drafts.

## Write field contracts

### Create GitHub issue

Resource: `/github/repos/{owner}/{repo}/issues/<id>.json`
Schema: `/github/repos/{owner}/{repo}/issues/.schema.json`
Create example: `/github/repos/{owner}/{repo}/issues/.create.example.json`
Required fields: `title`.
Optional fields: `body`, `labels`, `assignees`, `milestone`, `state`.

### Create GitHub issue comment

Resource: `/github/repos/{owner}/{repo}/issues/{issueNumber}/comments/<id>.json`
Schema: `/github/repos/{owner}/{repo}/issues/{issueNumber}/comments/.schema.json`
Create example: `/github/repos/{owner}/{repo}/issues/{issueNumber}/comments/.create.example.json`
Required fields: `body`.
Optional fields: none.

### Submit GitHub pull request review

Resource: `/github/repos/{owner}/{repo}/pulls/{pullNumber}/reviews/<id>.json`
Schema: `/github/repos/{owner}/{repo}/pulls/{pullNumber}/reviews/.schema.json`
Create example: `/github/repos/{owner}/{repo}/pulls/{pullNumber}/reviews/.create.example.json`
Required fields: `event`, `body`, `comments`.
Optional fields: `metadata`.

Fields:

- `event` (required, enum) - Review event to submit. Allowed values: `APPROVE`, `COMMENT`, `REQUEST_CHANGES`.
- `body` (required, string) - Top-level review body.
- `comments` (required, array) - Inline review comments. Use an empty array for a body-only review.
- `comments[].path` (required, string) - Repository-relative file path.
- `comments[].line` (required, integer) - Line number on the selected side.
- `comments[].side` (optional, enum) - Diff side for the comment. Defaults to RIGHT. Allowed values: `LEFT`, `RIGHT`.
- `comments[].body` (required, string) - Inline comment body.
- `comments[].suggestion` (optional, string) - Optional suggested replacement text appended to the comment body.
- `metadata` (optional, object) - Optional submission metadata.
- `metadata.commitSha` (optional, string) - Commit SHA to anchor the review to.
- `metadata.connectionId` (optional, string) - Relayfile connection id override.
- `metadata.providerConfigKey` (optional, string) - GitHub provider config key override.

## Create Examples
Read the resource `.schema.json` first, then use the sibling `.create.example.json` as a minimal create document. The example intentionally omits read-only fields.
