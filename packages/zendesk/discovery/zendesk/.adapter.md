# Zendesk adapter

The Zendesk adapter exposes tickets, users, and organizations under `/zendesk`, with writeback routes for creating tickets, ticket comments, and users.

Read-only mounts:
- `/zendesk/tickets/<ticketId>.json` - Ticket records.
- `/zendesk/tickets/<ticketId>/comments/<commentId>.json` - Ticket comment records.
- `/zendesk/users/<userId>.json` - User records.
- `/zendesk/organizations/<organizationId>.json` - Organization records.

Write endpoints:

| Path | Schema | What it does |
|---|---|---|
| `/zendesk/tickets/{ticketId}/comments/new.json` | `/zendesk/tickets/{ticketId}/comments/new.schema.json` | Adds a comment to a Zendesk ticket. |
| `/zendesk/tickets/new.json` | `/zendesk/tickets/new.schema.json` | Creates a Zendesk ticket. |
| `/zendesk/users/new.json` | `/zendesk/users/new.schema.json` | Creates a Zendesk user. |

## Write field contracts

### Create Zendesk ticket comment

Path: `/zendesk/tickets/{ticketId}/comments/new.json`
Schema: `/zendesk/tickets/{ticketId}/comments/new.schema.json`
Example: `/zendesk/tickets/{ticketId}/comments/new.example.json`
Required fields: `body`.
Optional fields: `html_body`, `public`.

Fields:

- `body` (required, string) - Plain-text comment body.
- `html_body` (optional, string) - HTML comment body.
- `public` (optional, boolean) - Whether the comment is public. Defaults to true.

### Create Zendesk ticket

Path: `/zendesk/tickets/new.json`
Schema: `/zendesk/tickets/new.schema.json`
Example: `/zendesk/tickets/new.example.json`
Required fields: `subject`.
Optional fields: `description`, `comment`, `requester_id`, `assignee_id`, `group_id`, `organization_id`, `priority`, `status`, `type`, `tags`, `custom_fields`.

Fields:

- `subject` (required, string) - Ticket subject.
- `description` (optional, string) - Ticket description.
- `comment` (optional, object) - Initial ticket comment.
- `requester_id` (optional, integer) - Requester user id.
- `assignee_id` (optional, integer) - Assignee user id.
- `group_id` (optional, integer) - Group id.
- `organization_id` (optional, integer) - Organization id.
- `priority` (optional, enum) - Ticket priority. Allowed values: `urgent`, `high`, `normal`, `low`.
- `status` (optional, enum) - Ticket status. Allowed values: `new`, `open`, `pending`, `hold`, `solved`, `closed`.
- `type` (optional, enum) - Ticket type. Allowed values: `problem`, `incident`, `question`, `task`.
- `tags` (optional, array) - Ticket tags.
- `custom_fields` (optional, array) - Zendesk custom fields.

### Create Zendesk user

Path: `/zendesk/users/new.json`
Schema: `/zendesk/users/new.schema.json`
Example: `/zendesk/users/new.example.json`
Required fields: `name`.
Optional fields: `email`, `role`, `phone`, `organization_id`, `external_id`, `tags`, `user_fields`, `verified`.

Fields:

- `name` (required, string) - User name.
- `email` (optional, string, email) - User email address.
- `role` (optional, string) - Zendesk user role.
- `phone` (optional, string) - User phone number.
- `organization_id` (optional, integer) - Organization id.
- `external_id` (optional, string) - External id.
- `tags` (optional, array) - User tags.
- `user_fields` (optional, object) - Zendesk user field values.
- `verified` (optional, boolean) - Whether the email is verified.

## How to use (agents)
1. Read the relevant `*.schema.json` to learn the required shape.
2. Optional: read the matching `new.example.json` for a starter document.
3. Write your document to the `new.json` path.
4. Read back the most-recently-created sibling record written by the adapter.
