# Zendesk adapter

The Zendesk adapter exposes tickets, users, and organizations under `/zendesk`, with writeback routes for creating tickets, ticket comments, and users.

Read-only mounts:
- `/zendesk/tickets/<ticketId>.json` - Ticket records.
- `/zendesk/tickets/<ticketId>/comments/<commentId>.json` - Ticket comment records.
- `/zendesk/users/<userId>.json` - User records.
- `/zendesk/organizations/<organizationId>.json` - Organization records.

Write endpoints:
| Path | Schema | What it does |
|---|---|---|
| `/zendesk/tickets/{ticketId}/comments/new.json` | `/zendesk/tickets/{ticketId}/comments/new.schema.json` | Adds a comment to a Zendesk ticket. |
| `/zendesk/tickets/new.json` | `/zendesk/tickets/new.schema.json` | Creates a Zendesk ticket. |
| `/zendesk/users/new.json` | `/zendesk/users/new.schema.json` | Creates a Zendesk user. |

## How to use (agents)
1. Read the relevant `*.schema.json` to learn the required shape.
2. Optional: read the matching `new.example.json` for a starter document.
3. Write your document to the `new.json` path.
4. Read back the most-recently-created sibling record written by the adapter.
