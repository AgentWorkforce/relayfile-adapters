# GitLab adapter

The GitLab adapter exposes projects, merge requests, discussions, issues, commits, pipelines, and jobs under `/gitlab`, with writeback routes for merge request discussions and issue notes.

Read-only mounts:
- `/gitlab/projects/<namespace>/<project>/merge_requests/<iid>/metadata.json` - Merge request metadata.
- `/gitlab/projects/<namespace>/<project>/merge_requests/<iid>/discussions/<discussionId>.json` - Merge request discussions.
- `/gitlab/projects/<namespace>/<project>/issues/<iid>/metadata.json` - Issue metadata.
- `/gitlab/projects/<namespace>/<project>/pipelines/<pipelineId>/jobs/<jobId>.json` - Pipeline job records.

Resources:

| Resource | Schema | Create example | ID pattern | What it does |
|---|---|---|---|---|
| `/gitlab/projects/{projectPath}/merge_requests/{mergeRequestIid}/discussions/<id>.json` | `/gitlab/projects/{projectPath}/merge_requests/{mergeRequestIid}/discussions/.schema.json` | `/gitlab/projects/{projectPath}/merge_requests/{mergeRequestIid}/discussions/.create.example.json` | `^[A-Za-z0-9_.:-]+$` | Creates a discussion on a merge request. |
| `/gitlab/projects/{projectPath}/issues/{issueIid}/comments/<id>.json` | `/gitlab/projects/{projectPath}/issues/{issueIid}/comments/.schema.json` | `/gitlab/projects/{projectPath}/issues/{issueIid}/comments/.create.example.json` | `^[A-Za-z0-9_.:-]+$` | Creates a note on an issue. |

## Operations

| To... | Do... |
|---|---|
| Read | `cat <id>.json` |
| Edit | Write a partial JSON object to `<id>.json`. Only included mutable fields PATCH; fields marked `readOnly` in `.schema.json` are rejected. |
| Create | Write JSON to any non-canonical filename such as `draft-title.json`. The adapter creates the record at `<real-id>.json` and rewrites the draft as `{ "created": "<real-id>", "path": "<resource>/<real-id>.json", "url": "<provider-url>" }`. |
| Delete | `rm <id>.json` for canonical ids. |

## ID Patterns
- `/gitlab/projects/{projectPath}/merge_requests/{mergeRequestIid}/discussions/<id>.json`: `^[A-Za-z0-9_.:-]+$`. Filenames that do not match this pattern are treated as create drafts.
- `/gitlab/projects/{projectPath}/issues/{issueIid}/comments/<id>.json`: `^[A-Za-z0-9_.:-]+$`. Filenames that do not match this pattern are treated as create drafts.

## Write field contracts

### Create GitLab merge request discussion

Resource: `/gitlab/projects/{projectPath}/merge_requests/{mergeRequestIid}/discussions/<id>.json`
Schema: `/gitlab/projects/{projectPath}/merge_requests/{mergeRequestIid}/discussions/.schema.json`
Create example: `/gitlab/projects/{projectPath}/merge_requests/{mergeRequestIid}/discussions/.create.example.json`
Required fields: `body`.
Optional fields: `position`, `created_at`.

Fields:

- `body` (required, string) - Markdown note body.
- `position` (optional, object) - Optional GitLab position object for diff discussions.
- `created_at` (optional, string, date-time) - Optional timestamp for imports when supported by GitLab.

### Create GitLab issue note

Resource: `/gitlab/projects/{projectPath}/issues/{issueIid}/comments/<id>.json`
Schema: `/gitlab/projects/{projectPath}/issues/{issueIid}/comments/.schema.json`
Create example: `/gitlab/projects/{projectPath}/issues/{issueIid}/comments/.create.example.json`
Required fields: `body`.
Optional fields: `position`, `created_at`.

Fields:

- `body` (required, string) - Markdown note body.
- `position` (optional, object) - Optional GitLab position object for diff discussions.
- `created_at` (optional, string, date-time) - Optional timestamp for imports when supported by GitLab.

## Create Examples
Read the resource `.schema.json` first, then use the sibling `.create.example.json` as a minimal create document. The example intentionally omits read-only fields.
