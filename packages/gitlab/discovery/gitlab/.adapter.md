# GitLab adapter

The GitLab adapter exposes projects, merge requests, discussions, issues, commits, pipelines, and jobs under `/gitlab`, with writeback routes for merge request discussions and issue notes.

Read-only mounts:
- `/gitlab/projects/<namespace>/<project>/merge_requests/<iid>/metadata.json` - Merge request metadata.
- `/gitlab/projects/<namespace>/<project>/merge_requests/<iid>/discussions/<discussionId>.json` - Merge request discussions.
- `/gitlab/projects/<namespace>/<project>/issues/<iid>/metadata.json` - Issue metadata.
- `/gitlab/projects/<namespace>/<project>/pipelines/<pipelineId>/jobs/<jobId>.json` - Pipeline job records.

Write endpoints:

| Path | Schema | What it does |
|---|---|---|
| `/gitlab/projects/{projectPath}/merge_requests/{mergeRequestIid}/discussions/new.json` | `/gitlab/projects/{projectPath}/merge_requests/{mergeRequestIid}/discussions/new.schema.json` | Creates a discussion on a merge request. |
| `/gitlab/projects/{projectPath}/issues/{issueIid}/comments/new.json` | `/gitlab/projects/{projectPath}/issues/{issueIid}/comments/new.schema.json` | Creates a note on an issue. |

## Write field contracts

### Create GitLab merge request discussion

Path: `/gitlab/projects/{projectPath}/merge_requests/{mergeRequestIid}/discussions/new.json`
Schema: `/gitlab/projects/{projectPath}/merge_requests/{mergeRequestIid}/discussions/new.schema.json`
Example: `/gitlab/projects/{projectPath}/merge_requests/{mergeRequestIid}/discussions/new.example.json`
Required fields: `body`.
Optional fields: `position`, `created_at`.

Fields:

- `body` (required, string) - Markdown note body.
- `position` (optional, object) - Optional GitLab position object for diff discussions.
- `created_at` (optional, string, date-time) - Optional timestamp for imports when supported by GitLab.

### Create GitLab issue note

Path: `/gitlab/projects/{projectPath}/issues/{issueIid}/comments/new.json`
Schema: `/gitlab/projects/{projectPath}/issues/{issueIid}/comments/new.schema.json`
Example: `/gitlab/projects/{projectPath}/issues/{issueIid}/comments/new.example.json`
Required fields: `body`.
Optional fields: `position`, `created_at`.

Fields:

- `body` (required, string) - Markdown note body.
- `position` (optional, object) - Optional GitLab position object for diff discussions.
- `created_at` (optional, string, date-time) - Optional timestamp for imports when supported by GitLab.

## How to use (agents)
1. Read the relevant `*.schema.json` to learn the required shape.
2. Optional: read the matching `new.example.json` for a starter document.
3. Write your document to the `new.json` path.
4. Read back the most-recently-created sibling record written by the adapter.
