# GitLab adapter

The GitLab adapter exposes projects, merge requests, discussions, issues, commits, pipelines, jobs, deployments, and tags under `/gitlab`. Directory records that own child files use `<id>__<slug>/meta.json`; flat records use `<slug>__<id>.json` or `<id>.json` when no useful slug exists.

Read-only mounts:
- `/gitlab/LAYOUT.md` - Provider-specific tree guide.
- `/gitlab/_index.json` - Top-level roots.
- `/gitlab/projects/_index.json` - Materialized projects.
- `/gitlab/projects/<namespace>/<project>/merge_requests/<iid>__<slug>/meta.json` - Merge request metadata.
- `/gitlab/projects/<namespace>/<project>/merge_requests/<iid>__<slug>/discussions/<discussionId>.json` - Merge request discussions.
- `/gitlab/projects/<namespace>/<project>/issues/<iid>__<slug>/meta.json` - Issue metadata.
- `/gitlab/projects/<namespace>/<project>/pipelines/<pipelineId>__<ref>/jobs/<jobId>.json` - Pipeline job records.

Aliases:
- `/gitlab/projects/<namespace>/<project>/merge_requests/by-id/<iid>.json`
- `/gitlab/projects/<namespace>/<project>/merge_requests/by-title/<slug>__<iid>.json`
- `/gitlab/projects/<namespace>/<project>/issues/by-id/<iid>.json`
- `/gitlab/projects/<namespace>/<project>/issues/by-title/<slug>__<iid>.json`
- `/gitlab/projects/<namespace>/<project>/pipelines/by-ref/<ref-slug>__<pipelineId>.json`
- `/gitlab/projects/<namespace>/<project>/pipelines/by-status/<status>/<pipelineId>.json`

Resources:

| Resource | Schema | Create example | ID pattern | What it does |
|---|---|---|---|---|
| `/gitlab/projects/{projectPath}/merge_requests/{mergeRequestIid}__{slug}/discussions/<id>.json` | `/gitlab/projects/{projectPath}/merge_requests/{mergeRequestIid}__{slug}/discussions/.schema.json` | `/gitlab/projects/{projectPath}/merge_requests/{mergeRequestIid}__{slug}/discussions/.create.example.json` | `^[A-Za-z0-9_.:-]+$` | Creates a discussion on a merge request. |
| `/gitlab/projects/{projectPath}/issues/{issueIid}__{slug}/comments/<id>.json` | `/gitlab/projects/{projectPath}/issues/{issueIid}__{slug}/comments/.schema.json` | `/gitlab/projects/{projectPath}/issues/{issueIid}__{slug}/comments/.create.example.json` | `^[A-Za-z0-9_.:-]+$` | Creates a note on an issue. |

## Operations

| To... | Do... |
|---|---|
| Read | `cat <id>.json` or follow a `by-*` pointer's `canonicalPath`. |
| Edit | Write a partial JSON object to the canonical `meta.json`; only included mutable fields PATCH. |
| Create | Write JSON to any non-canonical filename such as `create request.json`; `new.json` has no special meaning. |
| Delete | Remove canonical discussion or issue note files. |

## ID Patterns
- `/gitlab/projects/{projectPath}/merge_requests/{mergeRequestIid}__{slug}/discussions/<id>.json`: `^[A-Za-z0-9_.:-]+$`. Filenames that do not match this pattern are treated as create drafts.
- `/gitlab/projects/{projectPath}/issues/{issueIid}__{slug}/comments/<id>.json`: `^[A-Za-z0-9_.:-]+$`. Filenames that do not match this pattern are treated as create drafts.

## Write field contracts

### Create GitLab merge request discussion

Resource: `/gitlab/projects/{projectPath}/merge_requests/{mergeRequestIid}__{slug}/discussions/<id>.json`
Schema: `/gitlab/projects/{projectPath}/merge_requests/{mergeRequestIid}__{slug}/discussions/.schema.json`
Create example: `/gitlab/projects/{projectPath}/merge_requests/{mergeRequestIid}__{slug}/discussions/.create.example.json`
Required fields: `body`.
Optional fields: `position`, `created_at`.

Fields:

- `body` (required, string) - Markdown note body.
- `position` (optional, object) - Optional GitLab position object for diff discussions.
- `created_at` (optional, string, date-time) - Optional timestamp for imports when supported by GitLab.

### Create GitLab issue note

Resource: `/gitlab/projects/{projectPath}/issues/{issueIid}__{slug}/comments/<id>.json`
Schema: `/gitlab/projects/{projectPath}/issues/{issueIid}__{slug}/comments/.schema.json`
Create example: `/gitlab/projects/{projectPath}/issues/{issueIid}__{slug}/comments/.create.example.json`
Required fields: `body`.
Optional fields: `created_at`.

Fields:

- `body` (required, string) - Markdown note body.
- `created_at` (optional, string, date-time) - Optional timestamp for imports when supported by GitLab.

## Create Examples
Read the resource `.schema.json` first, then use the sibling `.create.example.json` as a minimal create document. The example intentionally omits read-only fields such as `id`, `updated_at`, `_webhook`, and `_connection`.
