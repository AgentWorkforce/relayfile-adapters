# SharePoint Storage Bridge Adapter

SharePoint maps Microsoft Graph subscriptions plus drive delta into Relayfile StorageBridgeEvent envelopes under /sharepoint/{siteId}/{driveId}/{path}. The adapter validates configuration, normalizes push notifications, publishes storage events, fetches content through injected provider clients, and resolves file-native writeback into provider-shaped operations.

## Read-only Mounts

Read-only synchronized content is mounted at `/sharepoint/{siteId}/{driveId}/{path}`. Deleted provider objects emit deletion events with null fetch content.

## Writable Resources

The resource `/sharepoint/{siteId}/{driveId}/items` ships `/sharepoint/{siteId}/{driveId}/items/.schema.json` and `/sharepoint/{siteId}/{driveId}/items/.create.example.json`.

The resource `/sharepoint/subscriptions` ships `/sharepoint/subscriptions/.schema.json` and `/sharepoint/subscriptions/.create.example.json`.

The static discovery resource `/sharepoint/items/.schema.json` ships `/sharepoint/items/.create.example.json`.

## Operations

Create writes use any non-canonical filename in the resource directory. Canonical filenames patch mutable fields only. Removing a canonical file resolves to provider delete operations. Read-only fields such as `id`, `createdAt`, `updatedAt`, `url`, `_webhook`, and `_connection` are rejected before provider calls.

The object resource resolves to `/v1.0/sites/{siteId}/drives/{driveId}/items/{itemId}`. The lifecycle resource resolves to `/v1.0/subscriptions`.

## Push and Auth Validation

Graph subscription validation tokens are surfaced by getWebhookChallenge; signed local callbacks use x-relayfile-signature until the deployment terminates Graph auth at the edge.

## Nango template

Nango anchor: `sharepoint-online`; sync `sharepoint-online-files`; scopes `Files.Read.All`, `Files.ReadWrite.All`, and `offline_access`.

## Nango Fallback

SharePoint declares `sharepoint-online-files` for scheduled fallback records using provider config key `sharepoint-online`.

## ID Patterns

`/sharepoint/{siteId}/{driveId}/items`: `^[A-Za-z0-9!._-]+$`.

`/sharepoint/subscriptions`: `^[A-Za-z0-9_-]+$`.
