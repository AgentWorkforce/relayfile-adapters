# Intercom adapter

The Intercom adapter exposes conversations, contacts, and companies under `/intercom`, with writeback routes for creating and updating those objects.

Read-only mounts:
- `/intercom/conversations/<conversationId>.json` - Conversation records.
- `/intercom/contacts/<contactId>.json` - Contact records.
- `/intercom/companies/<companyId>.json` - Company records.

Resources:

| Resource | Schema | Create example | ID pattern | What it does |
|---|---|---|---|---|
| `/intercom/conversations/<id>.json` | `/intercom/conversations/.schema.json` | `/intercom/conversations/.create.example.json` | `^[A-Za-z0-9_-]+$` | Creates an Intercom conversation. |
| `/intercom/contacts/<id>.json` | `/intercom/contacts/.schema.json` | `/intercom/contacts/.create.example.json` | `^[A-Za-z0-9_-]+$` | Creates an Intercom contact. |
| `/intercom/companies/<id>.json` | `/intercom/companies/.schema.json` | `/intercom/companies/.create.example.json` | `^[A-Za-z0-9_-]+$` | Creates an Intercom company. |

## Operations

| To... | Do... |
|---|---|
| Read | `cat <canonical-resource-path>` after listing the resource directory or following an alias when one is available. Use the resource table and ID patterns below to determine whether a resource uses a bare id, an adapter-specific slug/id filename, or an exact sidecar path such as `content.md`. |
| Edit | Write the resource update payload to the canonical resource path. For JSON resources, included mutable fields PATCH; fields marked `readOnly` in `.schema.json` are rejected. |
| Create | Write JSON to any non-canonical filename such as `create request.json`. The adapter creates the record at its canonical resource path and rewrites the draft as `{ "created": "<real-id>", "path": "<canonical-resource-path>", "url": "<provider-url>" }`. |
| Ignore | Editor scratch files named `partial.json`, `.tmp.json`, `.partial.json`, `*.tmp.json`, or `*.partial.json` are ignored and never treated as create drafts. |
| Delete | `rm <canonical-resource-path>` for canonical records. |

## ID Patterns
- `/intercom/conversations/<id>.json`: `^[A-Za-z0-9_-]+$`. Filenames that do not match this pattern are treated as create drafts.
- `/intercom/contacts/<id>.json`: `^[A-Za-z0-9_-]+$`. Filenames that do not match this pattern are treated as create drafts.
- `/intercom/companies/<id>.json`: `^[A-Za-z0-9_-]+$`. Filenames that do not match this pattern are treated as create drafts.

## Write field contracts

### Create Intercom conversation

Resource: `/intercom/conversations/<id>.json`
Schema: `/intercom/conversations/.schema.json`
Create example: `/intercom/conversations/.create.example.json`
Required fields: none at the top level.
Optional fields: `from`, `body`, `message_type`.

Fields:

- `from` (optional, object) - Message author. Include the shape required by Intercom for the selected source type.
- `body` (optional, string) - Conversation body.
- `message_type` (optional, string) - Conversation message type.

### Create Intercom contact

Resource: `/intercom/contacts/<id>.json`
Schema: `/intercom/contacts/.schema.json`
Create example: `/intercom/contacts/.create.example.json`
Required fields: none at the top level.
Optional fields: `role`, `email`, `name`, `external_id`, `phone`.

Fields:

- `role` (optional, string) - Contact role, such as user or lead.
- `email` (optional, string, email) - Contact email address.
- `name` (optional, string) - Contact display name.
- `external_id` (optional, string) - External id for upsert-style workflows.
- `phone` (optional, string) - Contact phone number.

### Create Intercom company

Resource: `/intercom/companies/<id>.json`
Schema: `/intercom/companies/.schema.json`
Create example: `/intercom/companies/.create.example.json`
Required fields: none at the top level.
Optional fields: `company_id`, `name`, `plan`, `website`, `monthly_spend`.

Fields:

- `company_id` (optional, string) - Stable company id.
- `name` (optional, string) - Company name.
- `plan` (optional, string) - Plan name.
- `website` (optional, string, uri) - Company website URL.
- `monthly_spend` (optional, number) - Monthly spend value.

## Create Examples
Read the resource `.schema.json` first, then use the sibling `.create.example.json` as a minimal create document. The example intentionally omits read-only fields.
