# Intercom adapter

The Intercom adapter exposes conversations, contacts, and companies under `/intercom`, with writeback routes for creating and updating those objects.

Read-only mounts:
- `/intercom/conversations/<conversationId>.json` - Conversation records.
- `/intercom/contacts/<contactId>.json` - Contact records.
- `/intercom/companies/<companyId>.json` - Company records.

Write endpoints:
| Path | Schema | What it does |
|---|---|---|
| `/intercom/conversations/new.json` | `/intercom/conversations/new.schema.json` | Creates an Intercom conversation. |
| `/intercom/contacts/new.json` | `/intercom/contacts/new.schema.json` | Creates an Intercom contact. |
| `/intercom/companies/new.json` | `/intercom/companies/new.schema.json` | Creates an Intercom company. |

## How to use (agents)
1. Read the relevant `*.schema.json` to learn the required shape.
2. Optional: read the matching `new.example.json` for a starter document.
3. Write your document to the `new.json` path.
4. Read back the most-recently-created sibling record written by the adapter.
