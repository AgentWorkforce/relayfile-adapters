# HubSpot adapter

The HubSpot adapter exposes CRM contacts, companies, deals, and tickets under `/hubspot`, with writeback routes for creating and updating those CRM objects.

Read-only mounts:
- `/hubspot/contacts/<contactId>.json` - Contact records.
- `/hubspot/companies/<companyId>.json` - Company records.
- `/hubspot/deals/<dealId>.json` - Deal records.
- `/hubspot/tickets/<ticketId>.json` - Ticket records.

Write endpoints:

| Path | Schema | What it does |
|---|---|---|
| `/hubspot/contacts/new.json` | `/hubspot/contacts/new.schema.json` | Creates a HubSpot contact. |
| `/hubspot/companies/new.json` | `/hubspot/companies/new.schema.json` | Creates a HubSpot company. |
| `/hubspot/deals/new.json` | `/hubspot/deals/new.schema.json` | Creates a HubSpot deal. |
| `/hubspot/tickets/new.json` | `/hubspot/tickets/new.schema.json` | Creates a HubSpot ticket. |

## Write field contracts

### Create HubSpot contact

Path: `/hubspot/contacts/new.json`
Schema: `/hubspot/contacts/new.schema.json`
Example: `/hubspot/contacts/new.example.json`
Required fields: none at the top level.
Optional fields: `properties`.
Validation: satisfy exactly one of 2 allowed payload shapes.

Fields:

- `properties` (optional, object) - HubSpot CRM property names and values. Read HubSpot object property metadata to discover object-specific keys.
- additional properties (optional, value) - Writable HubSpot CRM property value.

## How to use (agents)
1. Read the relevant `*.schema.json` to learn the required shape.
2. Optional: read the matching `new.example.json` for a starter document.
3. Write your document to the `new.json` path.
4. Read back the most-recently-created sibling record written by the adapter.
