# HubSpot adapter

The HubSpot adapter exposes CRM contacts, companies, deals, and tickets under `/hubspot`, with writeback routes for creating and updating those CRM objects.

Read-only mounts:
- `/hubspot/contacts/<contactId>.json` - Contact records.
- `/hubspot/companies/<companyId>.json` - Company records.
- `/hubspot/deals/<dealId>.json` - Deal records.
- `/hubspot/tickets/<ticketId>.json` - Ticket records.

Resources:

| Resource | Schema | Create example | ID pattern | What it does |
|---|---|---|---|---|
| `/hubspot/contacts/<id>.json` | `/hubspot/contacts/.schema.json` | `/hubspot/contacts/.create.example.json` | `^(?:[A-Za-z0-9_.~-]+--)?\d+$` | Creates a HubSpot contact. |
| `/hubspot/companies/<id>.json` | `/hubspot/companies/.schema.json` | `/hubspot/companies/.create.example.json` | `^(?:[A-Za-z0-9_.~-]+--)?\d+$` | Creates a HubSpot company. |

## Operations

| To... | Do... |
|---|---|
| Read | `cat <canonical-resource-path>` after listing the resource directory or following an alias when one is available. Use the resource table and ID patterns below to determine whether a resource uses a bare id, an adapter-specific slug/id filename, or an exact sidecar path such as `content.md`. |
| Edit | Write the resource update payload to the canonical resource path. For JSON resources, included mutable fields PATCH; fields marked `readOnly` in `.schema.json` are rejected. |
| Create | Write JSON to any non-canonical filename such as `create request.json`. The adapter creates the record at its canonical resource path and rewrites the draft as `{ "created": "<real-id>", "path": "<canonical-resource-path>", "url": "<provider-url>" }`. |
| Ignore | Editor scratch files named `partial.json`, `.tmp.json`, `.partial.json`, `*.tmp.json`, or `*.partial.json` are ignored and never treated as create drafts. |
| Delete | `rm <canonical-resource-path>` for canonical records. |

## ID Patterns
- `/hubspot/contacts/<id>.json`: `^(?:[A-Za-z0-9_.~-]+--)?\d+$`. Filenames that do not match this pattern are treated as create drafts.
- `/hubspot/companies/<id>.json`: `^(?:[A-Za-z0-9_.~-]+--)?\d+$`. Filenames that do not match this pattern are treated as create drafts.

## Write field contracts

### Create HubSpot contact

Resource: `/hubspot/contacts/<id>.json`
Schema: `/hubspot/contacts/.schema.json`
Create example: `/hubspot/contacts/.create.example.json`
Required fields: none at the top level.
Optional fields: `properties`.
Validation: satisfy exactly one of 2 allowed payload shapes.

Fields:

- `properties` (optional, object) - HubSpot CRM property names and values. Read HubSpot object property metadata to discover object-specific keys.
- additional properties (optional, value) - Writable HubSpot CRM property value.
