# Recall adapter

The Recall adapter exposes meeting recordings under `/recall`. Recording and transcript webhooks converge on the same canonical JSON record so consumers can watch one mount and read transcript content from `payload.transcript_text`.

Read-only mounts:
- `/recall/recordings/<recordingId>.json` - Canonical recording records.
- `/recall/recordings/by-id/<recordingId>.json` - Stable id lookup aliases.
- `/recall/recordings/by-day/YYYY-MM-DD/_index.json` - Daily indexes keyed by the recording update/completion date.

Resources:

| Resource | Schema | Create example | ID pattern | What it does |
|---|---|---|---|---|
| `/recall/recordings/<id>.json` | `/recall/recordings/.schema.json` | `/recall/recordings/.create.example.json` | `^[A-Za-z0-9_-]+$` | Represents a Recall recording, including `transcript_text` when transcript data is available. |

## Operations

| To... | Do... |
|---|---|
| Read | `cat <canonical-resource-path>` after listing `/recall/recordings` or following a `/recall/recordings/by-id` alias. |
| Edit | Not currently supported. Recall recordings are synced read-only records. |
| Create | Not currently supported through Relayfile writeback. SDK uploads are created through the cloud Recall action. |
| Delete | Not currently supported through Relayfile writeback. |

## ID Patterns
- `/recall/recordings/<id>.json`: `^[A-Za-z0-9_-]+$`.

## Record Shape

Each canonical record is a JSON wrapper:

```json
{
  "provider": "recall",
  "objectType": "recording",
  "objectId": "rec_123",
  "canonicalPath": "/recall/recordings/rec_123.json",
  "payload": {
    "id": "rec_123",
    "status": "done",
    "title": "Customer call",
    "updated_at": "2026-06-11T12:00:00.000Z",
    "transcript_text": "..."
  }
}
```

Transcript webhook payloads preserve their raw fields under `payload` and add a flattened `payload.transcript_text` string when the source payload contains transcript segments.

## Read Examples

```bash
ls /recall/recordings
jq '.payload.transcript_text' /recall/recordings/<recording-id>.json
jq '.[0:10] | map({ id, title, updated, status })' /recall/recordings/_index.json
jq '.canonicalPath' /recall/recordings/by-id/<recording-id>.json
```
