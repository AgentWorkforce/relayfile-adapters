# Amazon S3 Storage Bridge Adapter

Amazon S3 maps S3 Event Notifications delivered through SQS long polling into Relayfile StorageBridgeEvent envelopes under /s3/{bucket}/{key}. The adapter validates configuration, normalizes push notifications, publishes storage events, fetches content through injected provider clients, and resolves file-native writeback into provider-shaped operations.

## Read-only Mounts

Read-only synchronized content is mounted at `/s3/{bucket}/{key}`. Deleted provider objects emit deletion events with null fetch content.

## Writable Resources

The resource `/s3/{bucket}/objects` ships `/s3/{bucket}/objects/.schema.json` and `/s3/{bucket}/objects/.create.example.json`.

The resource `/s3/queues` ships `/s3/queues/.schema.json` and `/s3/queues/.create.example.json`.

The static discovery resource `/s3/objects/.schema.json` ships `/s3/objects/.create.example.json`.

## Operations

Create writes use any non-canonical filename in the resource directory. Canonical filenames patch mutable fields only. Removing a canonical file resolves to provider delete operations. Read-only fields such as `id`, `createdAt`, `updatedAt`, `url`, `_webhook`, and `_connection` are rejected before provider calls.

The object resource resolves to `s3:PutObject`. The lifecycle resource resolves to `sqs:ReceiveMessage`.

## Push and Auth Validation

SQS message authenticity is delegated to AWS IAM credentials used by the queue poller. Local queue callbacks can be signed with x-relayfile-signature HMAC.

## Nango template

No Nango anchor; spec source: Amazon S3 Object API and S3 event notifications delivered through SQS.

## Nango Fallback

Amazon S3 is not in the Nango scheduled fallback catalog for this wave; operators must rely on the real-time push path.

## ID Patterns

`/s3/{bucket}/objects`: `^.+$`.

`/s3/queues`: `^[A-Za-z0-9._-]+$`.
