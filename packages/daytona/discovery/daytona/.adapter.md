# Daytona adapter

The Daytona adapter exposes per-organization usage and quota snapshots under `/daytona/usage`, with read-only mounts for sandboxes, snapshots, and volumes that are materialized from Daytona webhook deliveries. All resources in this adapter are read-only; there are no writeback routes.

Read-only mounts:
- `/daytona/usage/<organizationId>.json` - Per-organization quota and usage snapshots, polled hourly from Daytona's `GET /organizations/{organizationId}/usage`. One record per organization, keyed by organizationId.
- `/daytona/usage/by-id/<organizationId>.json` - Stable id-keyed alias for each usage record. Alias files are materialized canonical mirrors containing `{ provider, objectType, objectId, canonicalPath, payload }`.
- `/daytona/sandboxes/<id>.json` - Sandbox records materialized from `sandbox.created` and `sandbox.state.updated` webhook events. Keyed by provider-assigned sandbox id.
- `/daytona/snapshots/<id>.json` - Snapshot records materialized from `snapshot.created`, `snapshot.state.updated`, and `snapshot.removed` webhook events. Keyed by provider-assigned snapshot id.
- `/daytona/volumes/<id>.json` - Volume records materialized from `volume.created` and `volume.state.updated` webhook events. Keyed by provider-assigned volume id.

Resources:

| Resource | Schema | Create example | ID pattern | What it does |
|---|---|---|---|---|
| `/daytona/usage/<organizationId>.json` | `discovery/daytona/usage/.schema.json` | `discovery/daytona/usage/.create.example.json` | `^[A-Za-z0-9_.:@-]+$` | Read-only per-organization usage snapshot. No create/edit/delete supported. |

## Operations

| To... | Do... |
|---|---|
| Read | `cat <canonical-resource-path>` after listing the resource directory or following an alias. Start with `ls /daytona/usage` or read `/daytona/usage/_index.json`. |
| Discover resources | Read `/daytona/_index.json` for the top-level directory listing, then `/daytona/usage/_index.json` for per-org usage rows. |
| Find by id | Read `/daytona/usage/by-id/<organizationId>.json` and follow `canonicalPath`. |

## ID Patterns

- `/daytona/usage/<organizationId>.json`: `^[A-Za-z0-9_.:@-]+$`. Filenames match the organizationId verbatim (URL-encoded).

## Discovery Index Paths

- `/daytona/_index.json` - Root index listing all resource directories.
- `/daytona/usage/_index.json` - Usage index with rows `{ id, title, updated, canonicalPath, organizationId }`.
