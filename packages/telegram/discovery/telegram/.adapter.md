# Telegram adapter

The Telegram adapter exposes bot configuration, chats, event-sourced messages, callback queries, inline queries, reactions, and raw update envelopes under `/telegram`. Writeback routes proxy Telegram Bot API actions for rich bot messages, reactions, callback answers, inline-query answers, commands, and menu buttons. Conversation history starts when the bot is connected and receiving updates; Telegram Bot API does not provide arbitrary historical backfill.

Read-only mounts:
- `/telegram/bot/config.json` - Bot identity, command/menu/description, webhook, and sync-mode snapshot.
- `/telegram/chats/<chatId>__<title>/meta.json` - Chat records seen or enriched by the bot.
- `/telegram/chats/<chatId>__<title>/messages/<messageId>/meta.json` - Captured message records.
- `/telegram/chats/<chatId>__<title>/messages/<messageId>/reactions/<updateId>.json` - Reaction update records.
- `/telegram/callback-queries/<callbackQueryId>.json` - Inline keyboard callback query records.
- `/telegram/inline-queries/<inlineQueryId>.json` - Inline-mode query records.
- `/telegram/updates/<updateId>.json` - Raw Telegram update envelopes captured by optional history sync.

## Operations

| To... | Do... |
|---|---|
| Read | `cat <canonical-resource-path>` after listing the resource directory or following an alias when one is available. Use the resource table and ID patterns below to determine whether a resource uses a bare id, an adapter-specific slug/id filename, or an exact sidecar path such as `content.md`. |
| Edit | Write the resource update payload to the canonical resource path. For JSON resources, included mutable fields PATCH; fields marked `readOnly` in `.schema.json` are rejected. |
| Create | Write JSON to any non-canonical filename such as `create request.json`. The adapter creates the record at its canonical resource path and rewrites the draft as `{ "created": "<real-id>", "path": "<canonical-resource-path>", "url": "<provider-url>" }`. |
| Ignore | Editor scratch files named `partial.json`, `.tmp.json`, `.partial.json`, `*.tmp.json`, or `*.partial.json` are ignored and never treated as create drafts. |
| Delete | `rm <canonical-resource-path>` for canonical records. |

## Write field contracts

### Send Telegram message

Resource: `/telegram/chats/{chatId}/messages/<id>.json`
Schema: `/telegram/chats/{chatId}/messages/.schema.json`
Create example: `/telegram/chats/{chatId}/messages/.create.example.json`
Required fields: none at the top level.
Optional fields: `text`, `rich_message`, `photo`, `audio`, `document`, `video`, `animation`, `voice`, `video_note`, `media`, `latitude`, `longitude`, `title`, `address`, `phone_number`, `first_name`, `question`, `options`, `checklist`, `emoji`, `caption`, `parse_mode`, `disable_web_page_preview`, `disable_notification`, `reply_to_message_id`, `reply_markup`, `message_thread_id`, `business_connection_id`, `idempotencyKey`.
Validation: provide at least one of `text`, `rich_message`, `photo`, `audio`, `document`, `video`, `animation`, `voice`, `video_note`, `media`, `latitude`, `longitude`, `phone_number`, `first_name`, `question`, `options`, `checklist`, `emoji`.

Fields:

- `text` (optional, string) - Message text. Required unless a richer Bot API content field is supplied.
- `rich_message` (optional, object) - InputRichMessage payload for `sendRichMessage`.
- `photo` (optional, string) - Photo file_id, HTTP URL, or upload reference for `sendPhoto`.
- `audio` (optional, string) - Audio file_id, HTTP URL, or upload reference for `sendAudio`.
- `document` (optional, string) - Document file_id, HTTP URL, or upload reference for `sendDocument`.
- `video` (optional, string) - Video file_id, HTTP URL, or upload reference for `sendVideo`.
- `animation` (optional, string) - Animation file_id, HTTP URL, or upload reference for `sendAnimation`.
- `voice` (optional, string) - Voice file_id, HTTP URL, or upload reference for `sendVoice`.
- `video_note` (optional, string) - Video note file_id or upload reference for `sendVideoNote`.
- `media` (optional, array) - InputMedia objects for `sendMediaGroup`.
- `latitude` (optional, number) - Latitude for `sendLocation` or `sendVenue`.
- `longitude` (optional, number) - Longitude for `sendLocation` or `sendVenue`.
- `title` (optional, string) - Venue title when using `sendVenue`.
- `address` (optional, string) - Venue address when using `sendVenue`.
- `phone_number` (optional, string) - Contact phone number when using `sendContact`.
- `first_name` (optional, string) - Contact first name when using `sendContact`.
- `question` (optional, string) - Poll question when using `sendPoll`.
- `options` (optional, array) - Poll options when using `sendPoll`.
- `checklist` (optional, object) - InputChecklist payload for `sendChecklist`.
- `emoji` (optional, string) - Dice emoji when using `sendDice`.
- `caption` (optional, string) - Optional caption for media send methods.
- `parse_mode` (optional, enum) - Telegram parse mode for `text`. Allowed values: `Markdown`, `MarkdownV2`, `HTML`.
- `disable_web_page_preview` (optional, boolean) - Whether Telegram should suppress link previews.
- `disable_notification` (optional, boolean) - Whether Telegram should send silently.
- `reply_to_message_id` (optional, integer) - Optional message id to reply to.
- `reply_markup` (optional, object) - InlineKeyboardMarkup, ReplyKeyboardMarkup, ReplyKeyboardRemove, or ForceReply payload.
- `message_thread_id` (optional, integer) - Forum topic thread id for supergroups with topics enabled.
- `business_connection_id` (optional, string) - Business connection id when sending on behalf of a business account.
- `idempotencyKey` (optional, string) - Optional client-supplied idempotency token. The writeback engine may use it to deduplicate repeated posts; it is never forwarded to Telegram.

### Set Telegram reaction

Resource: `/telegram/chats/{chatId}/messages/{messageId}/reactions/<id>.json`
Schema: `/telegram/chats/{chatId}/messages/{messageId}/reactions/.schema.json`
Create example: `/telegram/chats/{chatId}/messages/{messageId}/reactions/.create.example.json`
Required fields: `reaction`.
Optional fields: `is_big`.

Fields:

- `reaction` (required, array) - Reaction list to set.
- `is_big` (optional, boolean) - Whether Telegram should show a big reaction animation.

### Answer Telegram callback query

Resource: `/telegram/callback-queries/<id>.json`
Schema: `/telegram/callback-queries/.schema.json`
Create example: `/telegram/callback-queries/.create.example.json`
Required fields: `callback_query_id`.
Optional fields: `text`, `show_alert`, `url`, `cache_time`.

Fields:

- `callback_query_id` (required, string) - Callback query id from `/telegram/callback-queries`.
- `text` (optional, string) - Notification text to show to the user.
- `show_alert` (optional, boolean) - Whether to show an alert instead of a toast notification.
- `url` (optional, string, uri) - Optional URL opened by the client.
- `cache_time` (optional, integer) - Maximum cache time in seconds.

### Answer Telegram inline query

Resource: `/telegram/inline-queries/<id>.json`
Schema: `/telegram/inline-queries/.schema.json`
Create example: `/telegram/inline-queries/.create.example.json`
Required fields: `inline_query_id`, `results`.
Optional fields: `cache_time`, `is_personal`, `next_offset`, `button`.

Fields:

- `inline_query_id` (required, string) - Inline query id from `/telegram/inline-queries`.
- `results` (required, array) - Inline results to return.
- `cache_time` (optional, integer) - Maximum cache time in seconds.
- `is_personal` (optional, boolean) - Whether results are cached only for the requesting user.
- `next_offset` (optional, string) - Pagination offset for the next inline query page.
- `button` (optional, object) - Optional InlineQueryResultsButton object.

### Set Telegram bot commands

Resource: `/telegram/bot/commands/<id>.json`
Schema: `/telegram/bot/commands/.schema.json`
Create example: `/telegram/bot/commands/.create.example.json`
Required fields: `commands`.
Optional fields: `scope`, `language_code`.

Fields:

- `commands` (required, array) - Commands to expose in Telegram clients.
- `scope` (optional, object) - Optional BotCommandScope object.
- `language_code` (optional, string) - Optional ISO 639-1 language code.

### Set Telegram menu button

Resource: `/telegram/bot/menu-button/<id>.json`
Schema: `/telegram/bot/menu-button/.schema.json`
Create example: `/telegram/bot/menu-button/.create.example.json`
Required fields: `menu_button`.
Optional fields: `chat_id`.

Fields:

- `chat_id` (optional, string) - Optional chat id. Omit for default menu button.
- `menu_button` (required, object) - Telegram MenuButton object, such as `{ "type": "commands" }` or web_app.
