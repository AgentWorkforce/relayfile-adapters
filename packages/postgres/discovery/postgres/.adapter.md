# Postgres adapter

The Postgres adapter exposes rows and LISTEN channels with file-native writeback discovery.

Read-only mounts:
- `/postgres/<db>/<schema>/<table>/<pk>.json` - Postgres row records.

Resources:

| Resource | Schema | Create example | ID pattern | What it does |
|---|---|---|---|---|
| `/postgres/rows/<id>.json` | `/postgres/rows/.schema.json` | `/postgres/rows/.create.example.json` | `^[A-Za-z0-9_.:-]+$` | Creates a row. |
| `/postgres/listeners/<id>.json` | `/postgres/listeners/.schema.json` | `/postgres/listeners/.create.example.json` | `^[A-Za-z0-9_.:-]+$` | Registers a LISTEN channel. |

## Operations

| To... | Do... |
|---|---|
| Read | `cat <id>.json` |
| Edit | Write a partial JSON object to `<id>.json`. Only included mutable fields PATCH; fields marked `readOnly` in `.schema.json` are rejected. |
| Create | Write JSON to any non-canonical filename such as `create request.json`. The adapter creates the record at `<real-id>.json` and rewrites the draft as `{ "created": "<real-id>", "path": "<resource>/<real-id>.json", "url": "<provider-url>" }`. |
| Delete | `rm <id>.json` for canonical ids. |

## ID Patterns
- `/postgres/rows/<id>.json`: `^[A-Za-z0-9_.:-]+$`. Filenames that do not match this pattern are treated as create drafts.
- `/postgres/listeners/<id>.json`: `^[A-Za-z0-9_.:-]+$`. Filenames that do not match this pattern are treated as create drafts.

## Write field contracts

### Create Postgres row

Resource: `/postgres/rows/<id>.json`
Schema: `/postgres/rows/.schema.json`
Create example: `/postgres/rows/.create.example.json`
Required fields: `_op`, `primaryKey`, `row`.
Optional fields: none.

Fields:

- `_op` (required, string) - Operation.
- `primaryKey` (required, string) - Primary key column.
- `row` (required, object) - Row values.

### Create Postgres listener

Resource: `/postgres/listeners/<id>.json`
Schema: `/postgres/listeners/.schema.json`
Create example: `/postgres/listeners/.create.example.json`
Required fields: `channel`.
Optional fields: `table`.

Fields:

- `channel` (required, string) - LISTEN channel.
- `table` (optional, string) - Table.

## Create Examples
Read the resource `.schema.json` first, then use the sibling `.create.example.json` as a minimal create document. The example intentionally omits read-only fields.
