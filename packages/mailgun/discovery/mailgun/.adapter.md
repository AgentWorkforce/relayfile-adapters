# Mailgun adapter

The Mailgun adapter exposes domains, messages, mailing lists, and list members under `/mailgun`, with writeback routes for sending messages, creating and updating lists, and upserting list members.

Read-only mounts:
- `/mailgun/domains/<domain>/messages/<messageId>.json` - Message records.
- `/mailgun/lists/<listAddress>.json` - Mailing list records.
- `/mailgun/lists/<listAddress>/members/<memberAddress>.json` - Mailing list member records.

Resources:

| Resource | Schema | Create example | ID pattern | What it does |
|---|---|---|---|---|
| `/mailgun/domains/{domain}/messages/<id>.json` | `/mailgun/domains/{domain}/messages/.schema.json` | `/mailgun/domains/{domain}/messages/.create.example.json` | `^[A-Za-z0-9_.:-]+$` | Sends a message through the domain named by the path. |
| `/mailgun/lists/<id>.json` | `/mailgun/lists/.schema.json` | `/mailgun/lists/.create.example.json` | `^[^@\s]+@[^@\s]+$` | Creates a Mailgun mailing list. |
| `/mailgun/lists/{listAddress}/members/<id>.json` | `/mailgun/lists/{listAddress}/members/.schema.json` | `/mailgun/lists/{listAddress}/members/.create.example.json` | `^[^@\s]+@[^@\s]+$` | Adds or updates a member in the mailing list named by the path. |

## Operations

| To... | Do... |
|---|---|
| Read | `cat <canonical-resource-path>` after listing the resource directory or following an alias when one is available. Use the resource table and ID patterns below to determine whether a resource uses a bare id, an adapter-specific slug/id filename, or an exact sidecar path such as `content.md`. |
| Edit | Write the resource update payload to the canonical resource path. For JSON resources, included mutable fields PATCH; fields marked `readOnly` in `.schema.json` are rejected. |
| Create | Write JSON to any non-canonical filename such as `create request.json`. The adapter creates the record at its canonical resource path and rewrites the draft as `{ "created": "<real-id>", "path": "<canonical-resource-path>", "url": "<provider-url>" }`. |
| Ignore | Editor scratch files named `partial.json`, `.tmp.json`, `.partial.json`, `*.tmp.json`, or `*.partial.json` are ignored and never treated as create drafts. |
| Delete | `rm <canonical-resource-path>` for canonical records. |

## ID Patterns
- `/mailgun/domains/{domain}/messages/<id>.json`: `^[A-Za-z0-9_.:-]+$`. Filenames that do not match this pattern are treated as create drafts.
- `/mailgun/lists/<id>.json`: `^[^@\s]+@[^@\s]+$`. Filenames that do not match this pattern are treated as create drafts.
- `/mailgun/lists/{listAddress}/members/<id>.json`: `^[^@\s]+@[^@\s]+$`. Filenames that do not match this pattern are treated as create drafts.

## Write field contracts

### Send Mailgun message

Resource: `/mailgun/domains/{domain}/messages/<id>.json`
Schema: `/mailgun/domains/{domain}/messages/.schema.json`
Create example: `/mailgun/domains/{domain}/messages/.create.example.json`
Required fields: `from`, `to`, `subject`.
Optional fields: `text`, `html`, `cc`, `bcc`, `reply-to`, `o:tag`, `o:campaign`, `o:tracking`, `o:tracking-clicks`, `o:tracking-opens`, `variables`.
Validation: provide at least one of `text`, `html`.

Fields:

- `from` (required, string) - Sender email address.
- `to` (required, array) - Recipient email addresses. A single string is also accepted by the resolver.
- `subject` (required, string) - Message subject.
- `text` (optional, string) - Plain-text message body. Required when `html` is omitted.
- `html` (optional, string) - HTML message body. Required when `text` is omitted.
- `cc` (optional, string) - CC recipients.
- `bcc` (optional, string) - BCC recipients.
- `reply-to` (optional, string) - Reply-To address.
- `o:tag` (optional, string) - Mailgun tag.
- `o:campaign` (optional, string) - Mailgun campaign id.
- `o:tracking` (optional, string) - Tracking setting.
- `o:tracking-clicks` (optional, string) - Click tracking setting.
- `o:tracking-opens` (optional, string) - Open tracking setting.
- `variables` (optional, object) - User variable map copied as Mailgun `v:*` fields.

### Create Mailgun list

Resource: `/mailgun/lists/<id>.json`
Schema: `/mailgun/lists/.schema.json`
Create example: `/mailgun/lists/.create.example.json`
Required fields: `address`.
Optional fields: `name`, `description`, `access_level`, `reply_preference`.

Fields:

- `address` (required, string) - Mailgun list email address.
- `name` (optional, string) - List display name.
- `description` (optional, string) - List description.
- `access_level` (optional, enum) - Mailgun posting access level. Allowed values: `readonly`, `members`, `everyone`.
- `reply_preference` (optional, enum) - Mailgun reply preference. Allowed values: `list`, `sender`.

### Upsert Mailgun list member

Resource: `/mailgun/lists/{listAddress}/members/<id>.json`
Schema: `/mailgun/lists/{listAddress}/members/.schema.json`
Create example: `/mailgun/lists/{listAddress}/members/.create.example.json`
Required fields: `address`.
Optional fields: `name`, `vars`, `subscribed`, `upsert`.

Fields:

- `address` (required, string) - Member email address.
- `name` (optional, string) - Member display name.
- `vars` (optional, object) - Mailgun member variables.
- `subscribed` (optional, boolean) - Whether the member is subscribed.
- `upsert` (optional, boolean) - Whether Mailgun should upsert an existing member. Defaults to true.

## Create Examples
Read the resource `.schema.json` first, then use the sibling `.create.example.json` as a minimal create document. The example intentionally omits read-only fields.
