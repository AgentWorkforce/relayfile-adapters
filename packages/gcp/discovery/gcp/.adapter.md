# GCP adapter

The GCP adapter materializes GCP state into the VFS under `/gcp`, with read-only mounts for Cloud Run services, Cloud Monitoring alert policies, and the current Cloud Billing FinOps snapshot. The adapter is auth-agnostic and declarative: it contains no HTTP client and no credentials. Auth is supplied at runtime by a provider (e.g. `@relayfile/provider-nango` via a Nango GCP connection), which overrides the per-resource API host (baseUrl) per call. All resources in this adapter are read-only; there are no writeback routes.

Read-only mounts:
- `/gcp/run/services/<serviceName>.json` - Cloud Run service records, sourced from the Cloud Run Admin API `GET https://run.googleapis.com/v2/projects/{project}/locations/{location}/services` (and revisions). One record per service, keyed by service name.
- `/gcp/run/services/by-id/<serviceName>.json` - Stable id-keyed alias for each Cloud Run service.
- `/gcp/monitoring/alerts/<policyId>.json` - Cloud Monitoring alert policy records, sourced from `GET https://monitoring.googleapis.com/v3/projects/{project}/alertPolicies` plus firing incidents (delivered via Pub/Sub push subscription). One record per alert policy, keyed by policyId.
- `/gcp/monitoring/alerts/by-id/<policyId>.json` - Stable id-keyed alias for each alert policy.
- `/gcp/billing/current.json` - The single FinOps current-state file, sourced from the Cloud Billing API `GET https://cloudbilling.googleapis.com/v1/billingAccounts/{id}` and project billing info.

Resources:

| Resource | Schema | Create example | ID pattern | What it does |
|---|---|---|---|---|
| `/gcp/run/services/<serviceName>.json` | `discovery/gcp/cloud-run-services/.schema.json` | `discovery/gcp/cloud-run-services/.create.example.json` | `^[A-Za-z0-9_.:@-]+$` | Read-only Cloud Run service snapshot. No create/edit/delete supported. |
| `/gcp/monitoring/alerts/<policyId>.json` | `discovery/gcp/monitoring-alerts/.schema.json` | `discovery/gcp/monitoring-alerts/.create.example.json` | `^[A-Za-z0-9_.:@-]+$` | Read-only Cloud Monitoring alert policy snapshot, updated by Pub/Sub-delivered incidents. |
| `/gcp/billing/current.json` | `discovery/gcp/billing/.schema.json` | `discovery/gcp/billing/.create.example.json` | `^[A-Za-z0-9_.:@-]+$` | Read-only current Cloud Billing FinOps snapshot. |

## Operations

| To... | Do... |
|---|---|
| Read | `cat <canonical-resource-path>` after listing the resource directory or following an alias. Start with `ls /gcp/run/services` or read `/gcp/run/services/_index.json`. |
| Discover resources | Read `/gcp/_index.json` for the top-level directory listing, then the per-resource `_index.json`. |
| Find by id | Read `/gcp/run/services/by-id/<serviceName>.json` or `/gcp/monitoring/alerts/by-id/<policyId>.json` and follow `canonicalPath`. |

## ID Patterns

- `/gcp/run/services/<serviceName>.json`: `^[A-Za-z0-9_.:@-]+$`. Filename matches the service name verbatim (URL-encoded).
- `/gcp/monitoring/alerts/<policyId>.json`: `^[A-Za-z0-9_.:@-]+$`. Filename matches the policyId verbatim (URL-encoded).
- `/gcp/billing/current.json`: single fixed-name file.

## Discovery Index Paths

- `/gcp/_index.json` - Root index listing all resource directories.
- `/gcp/run/services/_index.json` - Cloud Run services index with rows `{ id, title, updated, canonicalPath }`.
- `/gcp/monitoring/alerts/_index.json` - Alert policies index with rows `{ id, title, updated, canonicalPath }`.
