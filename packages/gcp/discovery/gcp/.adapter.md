# GCP adapter

The GCP adapter materializes GCP state into the VFS under `/gcp`, with read-only mounts for Cloud Run services, Cloud Monitoring alert policies, the current Cloud Billing FinOps snapshot, and Error Reporting groups. The adapter is auth-agnostic and declarative: it contains no HTTP client and no credentials. Auth is supplied at runtime by a provider (e.g. `@relayfile/provider-nango` via a Nango GCP connection), which overrides the per-resource API host (baseUrl) per call. All resources in this adapter are read-only; there are no writeback routes.

Connection scope keys are `project`, `location`, and `billingAccountId`. `project` selects the GCP project for Cloud Run and Monitoring reads, `location` narrows regional Cloud Run services, and `billingAccountId` selects the billing account snapshot.

Read-only mounts:
- `/gcp/run/services/<serviceName>.json` - Cloud Run service records, sourced from the Cloud Run Admin API `GET https://run.googleapis.com/v2/projects/{project}/locations/{location}/services` (and revisions). One record per service, keyed by service name.
- `/gcp/run/services/by-id/<serviceName>.json` - Stable id-keyed reconciliation anchor for each Cloud Run service.
- `/gcp/run/services/by-region/<region-slug>/<serviceName>.json` - Region lookup alias for each Cloud Run service.
- `/gcp/run/services/by-status/<ready|not-ready>/<serviceName>.json` - Readiness lookup alias for each Cloud Run service.
- `/gcp/monitoring/alerts/<policyId>.json` - Cloud Monitoring alert policy records, sourced from `GET https://monitoring.googleapis.com/v3/projects/{project}/alertPolicies` plus firing incidents (delivered via Pub/Sub push subscription). One record per alert policy, keyed by policyId.
- `/gcp/monitoring/alerts/by-id/<policyId>.json` - Stable id-keyed reconciliation anchor for each alert policy.
- `/gcp/monitoring/alerts/by-title/<display-name-slug>-<hash>__<policyId>.json` - Display-name lookup alias for each alert policy. The hash comes from the shared alias collision helper.
- `/gcp/monitoring/alerts/by-state/<open|closed>/<policyId>.json` - Lifecycle lookup alias for currently firing vs resolved alert policies.
- `/gcp/billing/current.json` - The single FinOps current-state file, sourced from the Cloud Billing API `GET https://cloudbilling.googleapis.com/v1/billingAccounts/{id}` and project billing info.
- `/gcp/error-reporting/groups/<groupId>.json` - Error Reporting group records, sourced from the Error Reporting API `GET https://clouderrorreporting.googleapis.com/v1beta1/projects/{project}/groupStats`. One record per error group.
- `/gcp/error-reporting/groups/by-id/<groupId>.json` - Stable id-keyed reconciliation anchor for each Error Reporting group.
- `/gcp/error-reporting/groups/by-service/<service-slug>/<groupId>.json` - Service lookup alias for each Error Reporting group with service metadata.
- `/gcp/error-reporting/groups/by-status/<open|resolved|acknowledged>/<groupId>.json` - Resolution-status lookup alias for each Error Reporting group.

## Operations

| To... | Do... |
|---|---|
| Read | `cat <canonical-resource-path>` after listing the resource directory or following an alias. Start with `ls /gcp/run/services` or read `/gcp/run/services/_index.json`. |
| Discover resources | Read `/gcp/_index.json` for the top-level directory listing, then the per-resource `_index.json`. |
| Find by id | Read `/gcp/run/services/by-id/<serviceName>.json` or `/gcp/monitoring/alerts/by-id/<policyId>.json` and follow `canonicalPath`. |
| Find Error Reporting groups | Read `/gcp/error-reporting/groups/by-service/<service-slug>/` or `/gcp/error-reporting/groups/by-status/<status>/` and follow `canonicalPath`. |

## ID Patterns

- `/gcp/run/services/<serviceName>.json`: `^[A-Za-z0-9_.:@-]+$`. Filename matches the service name verbatim (URL-encoded).
- `/gcp/monitoring/alerts/<policyId>.json`: `^[A-Za-z0-9_.:@-]+$`. Filename matches the policyId verbatim (URL-encoded).
- `/gcp/billing/current.json`: single fixed-name file.
- `/gcp/error-reporting/groups/<groupId>.json`: `^[A-Za-z0-9_.:@-]+$`. Filename matches the groupId verbatim (URL-encoded).

## Discovery Index Paths

- `/gcp/_index.json` - Root index listing all resource directories.
- `/gcp/run/services/_index.json` - Cloud Run services index with rows `{ id, title, updated, canonicalPath }`.
- `/gcp/run/services/_index.json` also includes filterless fields such as `region`, `status`, and `ready` when available.
- `/gcp/monitoring/alerts/_index.json` - Alert policies index with rows `{ id, title, updated, canonicalPath }`.
- `/gcp/monitoring/alerts/_index.json` also includes filterless fields such as `state`, `firing`, and `enabled` when available.
- `/gcp/billing/_index.json` - Billing current-state index with rows `{ id, title, updated, canonicalPath }`.
- `/gcp/error-reporting/groups/_index.json` - Error Reporting groups index with rows `{ id, title, updated, canonicalPath }`.
- `/gcp/error-reporting/groups/_index.json` also includes filterless fields such as `service` and `resolutionStatus` when available.

All indexes are sorted by `updated` descending.
