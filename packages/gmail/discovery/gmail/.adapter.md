# Gmail Storage Bridge Adapter

Gmail maps Gmail Watch API plus Cloud Pub/Sub and users.history.list into Relayfile StorageBridgeEvent envelopes under /gmail/{account}/threads/{threadId}.json. The adapter validates configuration, normalizes push notifications, publishes storage events, fetches content through injected provider clients, and resolves file-native writeback into provider-shaped operations.

## Read-only Mounts

Read-only synchronized content is mounted at `/gmail/{account}/threads/{threadId}.json`. Deleted provider objects emit deletion events with null fetch content.

## Writable Resources

The resource `/gmail/{account}/threads` ships `/gmail/{account}/threads/.schema.json` and `/gmail/{account}/threads/.create.example.json`.

The resource `/gmail/{account}/drafts` ships `/gmail/{account}/drafts/.schema.json` and `/gmail/{account}/drafts/.create.example.json`.

The resource `/gmail/watches` ships `/gmail/watches/.schema.json` and `/gmail/watches/.create.example.json`.

The static discovery resource `/gmail/threads/.schema.json` ships `/gmail/threads/.create.example.json`.

The static discovery resource `/gmail/drafts/.schema.json` ships `/gmail/drafts/.create.example.json`.

## Operations

Create writes use any non-canonical filename in the resource directory. Canonical filenames patch mutable fields only. Removing a canonical file resolves to provider delete operations. Read-only fields such as `id`, `createdAt`, `updatedAt`, `url`, `_webhook`, and `_connection` are rejected before provider calls.

The object resource resolves to `/gmail/v1/users/{account}/messages/{messageId}/modify`. The lifecycle resource resolves to `/gmail/v1/users/{account}/watch`.

## Push and Auth Validation

Cloud Pub/Sub delivery identity should be validated at the hosting layer; local receivers use x-relayfile-signature HMAC. History ids are normalized into thread resource events.

## Nango template

Nango anchor: `google-mail`; sync `google-mail-emails`; scopes `https://www.googleapis.com/auth/gmail.readonly` and `https://www.googleapis.com/auth/gmail.modify`.

## Nango Fallback

Gmail declares `google-mail-emails` for scheduled fallback records using provider config key `google-mail`.

## ID Patterns

`/gmail/{account}/threads`: `^[A-Za-z0-9_-]+$`.

`/gmail/{account}/drafts`: `^[A-Za-z0-9_-]+$`.

`/gmail/watches`: `^[^/]+$`.
