# Gmail adapter

The Gmail adapter exposes threads, drafts, and watches with file-native writeback discovery.

Read-only mounts:
- `/gmail/<account>/threads/<threadId>.json` - Gmail thread records.

Resources:

| Resource | Schema | Create example | ID pattern | What it does |
|---|---|---|---|---|
| `/gmail/threads/<id>.json` | `/gmail/threads/.schema.json` | `/gmail/threads/.create.example.json` | `^[A-Za-z0-9_.:-]+$` | Creates or updates a Gmail thread record. |
| `/gmail/drafts/<id>.json` | `/gmail/drafts/.schema.json` | `/gmail/drafts/.create.example.json` | `^[A-Za-z0-9_.:-]+$` | Creates a Gmail draft. |
| `/gmail/watches/<id>.json` | `/gmail/watches/.schema.json` | `/gmail/watches/.create.example.json` | `^[A-Za-z0-9_.:-]+$` | Starts a Gmail watch. |

## Operations

| To... | Do... |
|---|---|
| Read | `cat <id>.json` |
| Edit | Write a partial JSON object to `<id>.json`. Only included mutable fields PATCH; fields marked `readOnly` in `.schema.json` are rejected. |
| Create | Write JSON to any non-canonical filename such as `create request.json`. The adapter creates the record at `<real-id>.json` and rewrites the draft as `{ "created": "<real-id>", "path": "<resource>/<real-id>.json", "url": "<provider-url>" }`. |
| Delete | `rm <id>.json` for canonical ids. |

## ID Patterns
- `/gmail/threads/<id>.json`: `^[A-Za-z0-9_.:-]+$`. Filenames that do not match this pattern are treated as create drafts.
- `/gmail/drafts/<id>.json`: `^[A-Za-z0-9_.:-]+$`. Filenames that do not match this pattern are treated as create drafts.
- `/gmail/watches/<id>.json`: `^[A-Za-z0-9_.:-]+$`. Filenames that do not match this pattern are treated as create drafts.

## Write field contracts

### Create Gmail thread marker

Resource: `/gmail/threads/<id>.json`
Schema: `/gmail/threads/.schema.json`
Create example: `/gmail/threads/.create.example.json`
Required fields: `id`.
Optional fields: `labelIds`.

Fields:

- `id` (required, string) - Thread id.
- `labelIds` (optional, array) - Labels.

### Create Gmail draft

Resource: `/gmail/drafts/<id>.json`
Schema: `/gmail/drafts/.schema.json`
Create example: `/gmail/drafts/.create.example.json`
Required fields: `message`.
Optional fields: none.

Fields:

- `message` (required, object) - Draft message.

### Create Gmail watch

Resource: `/gmail/watches/<id>.json`
Schema: `/gmail/watches/.schema.json`
Create example: `/gmail/watches/.create.example.json`
Required fields: `topicName`.
Optional fields: `labelIds`.

Fields:

- `topicName` (required, string) - Pub/Sub topic.
- `labelIds` (optional, array) - Labels.

## Create Examples
Read the resource `.schema.json` first, then use the sibling `.create.example.json` as a minimal create document. The example intentionally omits read-only fields.
