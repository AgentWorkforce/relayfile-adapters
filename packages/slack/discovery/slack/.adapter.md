# Slack adapter

The Slack adapter exposes channels, users, messages, threads, replies, files, and reactions under `/slack`, with writeback routes for posting messages, replies, and reactions.

Read-only mounts:
- `/slack/channels/<channelId>.json` - Channel records.
- `/slack/channels/<channelId>/messages/<messageTs>.json` - Message records.
- `/slack/channels/<channelId>/messages/<messageTs>/replies/<replyTs>.json` - Thread reply records.
- `/slack/users/<userId>.json` - User records.

Write endpoints:
| Path | Schema | What it does |
|---|---|---|
| `/slack/channels/{channelId}/messages/new.json` | `/slack/channels/{channelId}/messages/new.schema.json` | Posts a top-level Slack message. |
| `/slack/channels/{channelId}/messages/{messageTs}/replies/new.json` | `/slack/channels/{channelId}/messages/{messageTs}/replies/new.schema.json` | Posts a reply in a Slack thread. |
| `/slack/channels/{channelId}/messages/{messageTs}/reactions/new.json` | `/slack/channels/{channelId}/messages/{messageTs}/reactions/new.schema.json` | Adds an emoji reaction to a Slack message. |

## How to use (agents)
1. Read the relevant `*.schema.json` to learn the required shape.
2. Optional: read the matching `new.example.json` for a starter document.
3. Write your document to the `new.json` path.
4. Read back the most-recently-created sibling record written by the adapter.
