# Slack adapter

The Slack adapter exposes channels, users, messages, threads, replies, files, and reactions under `/slack`, with writeback routes for posting messages, replies, direct messages, and reactions.

Read-only mounts:
- `/slack/channels/<channelId>.json` - Channel records.
- `/slack/channels/<channelId>/messages/<messageTs>.json` - Message records.
- `/slack/channels/<channelId>/messages/<messageTs>/replies/<replyTs>.json` - Thread reply records.
- `/slack/users/<userId>.json` - User records.

Resources:

| Resource | Schema | Create example | ID pattern | What it does |
|---|---|---|---|---|
| `/slack/channels/{channelId}/messages/<id>.json` | `/slack/channels/{channelId}/messages/.schema.json` | `/slack/channels/{channelId}/messages/.create.example.json` | `^[A-Za-z0-9_.:-]+(?:--[A-Za-z0-9_.:-]+)*$` | Posts a top-level Slack message. |
| `/slack/users/{userId}/messages/<id>.json` | `/slack/users/{userId}/messages/.schema.json` | `/slack/users/{userId}/messages/.create.example.json` | `^[A-Za-z0-9_.:-]+(?:--[A-Za-z0-9_.:-]+)*$` | Opens a Slack direct message and posts a message. |
| `/slack/channels/{channelId}/messages/{messageTs}/replies/<id>.json` | `/slack/channels/{channelId}/messages/{messageTs}/replies/.schema.json` | `/slack/channels/{channelId}/messages/{messageTs}/replies/.create.example.json` | `^[A-Za-z0-9_.:-]+(?:--[A-Za-z0-9_.:-]+)*$` | Posts a reply in a Slack thread. |
| `/slack/channels/{channelId}/messages/{messageTs}/reactions/<id>.json` | `/slack/channels/{channelId}/messages/{messageTs}/reactions/.schema.json` | `/slack/channels/{channelId}/messages/{messageTs}/reactions/.create.example.json` | `^[A-Za-z0-9_.:-]+(?:--[A-Za-z0-9_.:-]+)*$` | Adds an emoji reaction to a Slack message. |

## Operations

| To... | Do... |
|---|---|
| Read | `cat <id>.json` |
| Edit | Write a partial JSON object to `<id>.json`. Only included mutable fields PATCH; fields marked `readOnly` in `.schema.json` are rejected. |
| Create | Write JSON to any non-canonical filename such as `create request.json`. The adapter creates the record at `<real-id>.json` and rewrites the draft as `{ "created": "<real-id>", "path": "<resource>/<real-id>.json", "url": "<provider-url>" }`. |
| Delete | `rm <id>.json` for canonical ids. |

## ID Patterns
- `/slack/channels/{channelId}/messages/<id>.json`: `^[A-Za-z0-9_.:-]+(?:--[A-Za-z0-9_.:-]+)*$`. Filenames that do not match this pattern are treated as create drafts.
- `/slack/users/{userId}/messages/<id>.json`: `^[A-Za-z0-9_.:-]+(?:--[A-Za-z0-9_.:-]+)*$`. Filenames that do not match this pattern are treated as create drafts.
- `/slack/channels/{channelId}/messages/{messageTs}/replies/<id>.json`: `^[A-Za-z0-9_.:-]+(?:--[A-Za-z0-9_.:-]+)*$`. Filenames that do not match this pattern are treated as create drafts.
- `/slack/channels/{channelId}/messages/{messageTs}/reactions/<id>.json`: `^[A-Za-z0-9_.:-]+(?:--[A-Za-z0-9_.:-]+)*$`. Filenames that do not match this pattern are treated as create drafts.

## Write field contracts

### Post Slack message

Resource: `/slack/channels/{channelId}/messages/<id>.json`
Schema: `/slack/channels/{channelId}/messages/.schema.json`
Create example: `/slack/channels/{channelId}/messages/.create.example.json`
Required fields: none at the top level.
Optional fields: `text`, `blocks`, `attachments`, `channel`, `thread_ts`, `username`, `icon_emoji`, `icon_url`, `unfurl_links`, `unfurl_media`, `mrkdwn`.
Validation: provide at least one of `text`, `blocks`, `attachments`.

Fields:

- `text` (optional, string) - Message text. Required unless blocks or attachments are supplied.
- `blocks` (optional, array) - Slack Block Kit blocks.
- `attachments` (optional, array) - Slack message attachments.
- `channel` (optional, string) - Optional Slack channel id override.
- `thread_ts` (optional, string) - Optional Slack thread timestamp override.
- `username` (optional, string) - Bot display name override for tokens that support it.
- `icon_emoji` (optional, string) - Bot emoji icon override.
- `icon_url` (optional, string, uri) - Bot icon URL override.
- `unfurl_links` (optional, boolean) - Whether Slack should unfurl links.
- `unfurl_media` (optional, boolean) - Whether Slack should unfurl media.
- `mrkdwn` (optional, boolean) - Whether Slack should parse mrkdwn in text.

Validation: provide at least one of `text`, `blocks`, `attachments`.

### Post Slack thread reply

Resource: `/slack/channels/{channelId}/messages/{messageTs}/replies/<id>.json`
Schema: `/slack/channels/{channelId}/messages/{messageTs}/replies/.schema.json`
Create example: `/slack/channels/{channelId}/messages/{messageTs}/replies/.create.example.json`
Required fields: none at the top level.
Optional fields: `text`, `blocks`, `attachments`, `channel`, `thread_ts`, `username`, `icon_emoji`, `icon_url`, `unfurl_links`, `unfurl_media`, `mrkdwn`, `reply_broadcast`.
Validation: provide at least one of `text`, `blocks`, `attachments`.

Fields:

- `text` (optional, string) - Message text. Required unless blocks or attachments are supplied.
- `blocks` (optional, array) - Slack Block Kit blocks.
- `attachments` (optional, array) - Slack message attachments.
- `channel` (optional, string) - Optional Slack channel id override.
- `thread_ts` (optional, string) - Optional Slack thread timestamp override.
- `username` (optional, string) - Bot display name override for tokens that support it.
- `icon_emoji` (optional, string) - Bot emoji icon override.
- `icon_url` (optional, string, uri) - Bot icon URL override.
- `unfurl_links` (optional, boolean) - Whether Slack should unfurl links.
- `unfurl_media` (optional, boolean) - Whether Slack should unfurl media.
- `mrkdwn` (optional, boolean) - Whether Slack should parse mrkdwn in text.
- `reply_broadcast` (optional, boolean) - Whether Slack should also broadcast the reply to the channel.

### Add Slack reaction

Resource: `/slack/channels/{channelId}/messages/{messageTs}/reactions/<id>.json`
Schema: `/slack/channels/{channelId}/messages/{messageTs}/reactions/.schema.json`
Create example: `/slack/channels/{channelId}/messages/{messageTs}/reactions/.create.example.json`
Required fields: none at the top level.
Optional fields: `name`, `reaction`, `channel`.
Validation: provide at least one of `name`, `reaction`.

Fields:

- `name` (optional, string) - Emoji name without surrounding colons. `reaction` is also accepted.
- `reaction` (optional, string) - Alias for `name`.
- `channel` (optional, string) - Optional Slack channel id override.
