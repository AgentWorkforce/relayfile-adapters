# Slack adapter

The Slack adapter exposes channels, users, messages, threads, replies, files, and reactions under `/slack`, with writeback routes for posting channel messages, direct messages, replies, and reactions. Direct messages use `/slack/users/<userId>/messages` as the product contract for both reads and writes; Slack internal `D...` IM channel ids stay in record payload metadata. It also advertises history-independent discovery lookup indexes for Slack channel and user ids under `/discovery/slack`.

Read-only mounts:
- `/slack/channels/<channelId>.json` - Channel records.
- `/slack/channels/<channelId>/messages/<messageTs>/meta.json` - Message records.
- `/slack/channels/<channelId>/threads/<threadTs>/replies/<replyTs>/meta.json` - Thread reply records (directory records; legacy flat `/messages/<messageTs>/replies/<replyTs>.json` writeback paths still route for edits/deletes).
- `/slack/users/<userId>.json` - User records.
- `/slack/users/<userId>/messages/<messageTs>/meta.json` - 1:1 direct message records addressed by bare Slack user id.
- `/slack/users/<userId>/messages/<messageTs>/replies/<replyTs>/meta.json` - Threaded replies in a 1:1 direct message (directory records; legacy flat `<replyTs>.json` still readable).
- `/discovery/slack/channels/_index.json` - History-independent channel id/name lookup rows for writeback context, materialized from Slack channel discovery syncs.
- `/discovery/slack/users/_index.json` - History-independent user id/name lookup rows for direct-message writeback context, materialized from Slack user discovery syncs.

## Operations

| To... | Do... |
|---|---|
| Read | `cat <canonical-resource-path>` after listing the resource directory or following an alias when one is available. ## ID Patterns
- `/slack/channels/{channelId}/messages/<id>.json`: `^(?:meta|(?:[A-Za-z0-9_.:-]+--)?\d{10,}(?:_\d+)?)$`. Filenames that do not match this pattern are treated as create drafts.
- `/slack/users/{userId}/messages/<id>.json`: `^$`. Filenames that do not match this pattern are treated as create drafts.
- `/slack/channels/{channelId}/messages/{messageTs}/replies/<id>.json`: `^(?:[A-Za-z0-9_.:-]+--)?\d{10,}(?:_\d+)?$`. Filenames that do not match this pattern are treated as create drafts.
- `/slack/channels/{channelId}/messages/{messageTs}/reactions/<id>.json`: `^[A-Za-z0-9_.:-]+(?:--[A-Za-z0-9_.:-]+)*$`. Filenames that do not match this pattern are treated as create drafts.

## Write field contracts

### Post Slack message

Resource: `/slack/channels/{channelId}/messages/<id>.json`
Schema: `/slack/channels/{channelId}/messages/.schema.json`
Create example: `/slack/channels/{channelId}/messages/.create.example.json`
Required fields: none at the top level.
Optional fields: `text`, `blocks`, `attachments`, `channel`, `thread_ts`, `username`, `icon_emoji`, `icon_url`, `unfurl_links`, `unfurl_media`, `mrkdwn`, `idempotencyKey`, `parentRef`.
Validation: provide at least one of `text`, `blocks`, `attachments`.

Fields:

- `text` (optional, string) - Message text. Required unless blocks or attachments are supplied.
- `blocks` (optional, array) - Slack Block Kit blocks.
- `attachments` (optional, array) - Slack message attachments.
- `channel` (optional, string) - Optional Slack channel id override.
- `thread_ts` (optional, string) - Optional Slack thread timestamp override.
- `username` (optional, string) - Bot display name override for tokens that support it.
- `icon_emoji` (optional, string) - Bot emoji icon override.
- `icon_url` (optional, string, uri) - Bot icon URL override.
- `unfurl_links` (optional, boolean) - Whether Slack should unfurl links.
- `unfurl_media` (optional, boolean) - Whether Slack should unfurl media.
- `mrkdwn` (optional, boolean) - Whether Slack should parse mrkdwn in text.
- `idempotencyKey` (optional, string) - Optional client-supplied idempotency token. The writeback engine uses it to deduplicate repeated posts (e.g. across scheduled-tick re-runs); it is never forwarded to Slack.
- `parentRef` (optional, string) - Server-side threading: relay path of a parent message draft (e.g. the ref returned by a prior post) to thread this message under. The cloud orders this write after the parent delivers and sets thread_ts to its delivered timestamp; routing-only, never forwarded to Slack.

### Add Slack reaction

Resource: `/slack/channels/{channelId}/messages/{messageTs}/reactions/<id>.json`
Schema: `/slack/channels/{channelId}/messages/{messageTs}/reactions/.schema.json`
Create example: `/slack/channels/{channelId}/messages/{messageTs}/reactions/.create.example.json`
Required fields: none at the top level.
Optional fields: `name`, `reaction`, `channel`.
Validation: provide at least one of `name`, `reaction`.

Fields:

- `name` (optional, string) - Emoji name without surrounding colons. `reaction` is also accepted.
- `reaction` (optional, string) - Alias for `name`.
- `channel` (optional, string) - Optional Slack channel id override.
