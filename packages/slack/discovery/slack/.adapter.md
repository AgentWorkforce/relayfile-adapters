# Slack adapter

The Slack adapter exposes channels, users, messages, threads, replies, files, and reactions under `/slack`, with writeback routes for posting messages, replies, and reactions.

Read-only mounts:
- `/slack/channels/<channelId>.json` - Channel records.
- `/slack/channels/<channelId>/messages/<messageTs>.json` - Message records.
- `/slack/channels/<channelId>/messages/<messageTs>/replies/<replyTs>.json` - Thread reply records.
- `/slack/users/<userId>.json` - User records.

Write endpoints:

| Path | Schema | What it does |
|---|---|---|
| `/slack/channels/{channelId}/messages/new.json` | `/slack/channels/{channelId}/messages/new.schema.json` | Posts a top-level Slack message. |
| `/slack/channels/{channelId}/messages/{messageTs}/replies/new.json` | `/slack/channels/{channelId}/messages/{messageTs}/replies/new.schema.json` | Posts a reply in a Slack thread. |
| `/slack/channels/{channelId}/messages/{messageTs}/reactions/new.json` | `/slack/channels/{channelId}/messages/{messageTs}/reactions/new.schema.json` | Adds an emoji reaction to a Slack message. |

## Write field contracts

### Post Slack message

Path: `/slack/channels/{channelId}/messages/new.json`
Schema: `/slack/channels/{channelId}/messages/new.schema.json`
Example: `/slack/channels/{channelId}/messages/new.example.json`
Required fields: none at the top level.
Optional fields: `text`, `blocks`, `attachments`, `channel`, `thread_ts`, `username`, `icon_emoji`, `icon_url`, `unfurl_links`, `unfurl_media`, `mrkdwn`.
Validation: provide at least one of `text`, `blocks`, `attachments`.

Fields:

- `text` (optional, string) - Message text. Required unless blocks or attachments are supplied.
- `blocks` (optional, array) - Slack Block Kit blocks.
- `attachments` (optional, array) - Slack message attachments.
- `channel` (optional, string) - Optional Slack channel id override.
- `thread_ts` (optional, string) - Optional Slack thread timestamp override.
- `username` (optional, string) - Bot display name override for tokens that support it.
- `icon_emoji` (optional, string) - Bot emoji icon override.
- `icon_url` (optional, string, uri) - Bot icon URL override.
- `unfurl_links` (optional, boolean) - Whether Slack should unfurl links.
- `unfurl_media` (optional, boolean) - Whether Slack should unfurl media.
- `mrkdwn` (optional, boolean) - Whether Slack should parse mrkdwn in text.

### Post Slack thread reply

Path: `/slack/channels/{channelId}/messages/{messageTs}/replies/new.json`
Schema: `/slack/channels/{channelId}/messages/{messageTs}/replies/new.schema.json`
Example: `/slack/channels/{channelId}/messages/{messageTs}/replies/new.example.json`
Required fields: none at the top level.
Optional fields: `text`, `blocks`, `attachments`, `channel`, `thread_ts`, `username`, `icon_emoji`, `icon_url`, `unfurl_links`, `unfurl_media`, `mrkdwn`, `reply_broadcast`.
Validation: provide at least one of `text`, `blocks`, `attachments`.

Fields:

- `text` (optional, string) - Message text. Required unless blocks or attachments are supplied.
- `blocks` (optional, array) - Slack Block Kit blocks.
- `attachments` (optional, array) - Slack message attachments.
- `channel` (optional, string) - Optional Slack channel id override.
- `thread_ts` (optional, string) - Optional Slack thread timestamp override.
- `username` (optional, string) - Bot display name override for tokens that support it.
- `icon_emoji` (optional, string) - Bot emoji icon override.
- `icon_url` (optional, string, uri) - Bot icon URL override.
- `unfurl_links` (optional, boolean) - Whether Slack should unfurl links.
- `unfurl_media` (optional, boolean) - Whether Slack should unfurl media.
- `mrkdwn` (optional, boolean) - Whether Slack should parse mrkdwn in text.
- `reply_broadcast` (optional, boolean) - Whether Slack should also broadcast the reply to the channel.

### Add Slack reaction

Path: `/slack/channels/{channelId}/messages/{messageTs}/reactions/new.json`
Schema: `/slack/channels/{channelId}/messages/{messageTs}/reactions/new.schema.json`
Example: `/slack/channels/{channelId}/messages/{messageTs}/reactions/new.example.json`
Required fields: none at the top level.
Optional fields: `name`, `reaction`, `channel`.
Validation: provide at least one of `name`, `reaction`.

Fields:

- `name` (optional, string) - Emoji name without surrounding colons. `reaction` is also accepted.
- `reaction` (optional, string) - Alias for `name`.
- `channel` (optional, string) - Optional Slack channel id override.

## How to use (agents)
1. Read the relevant `*.schema.json` to learn the required shape.
2. Optional: read the matching `new.example.json` for a starter document.
3. Write your document to the `new.json` path.
4. Read back the most-recently-created sibling record written by the adapter.
