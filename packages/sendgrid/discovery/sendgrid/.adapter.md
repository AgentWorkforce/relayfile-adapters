# SendGrid adapter

The SendGrid adapter exposes sent mail and marketing contacts under `/sendgrid`, with writeback routes for sending mail and upserting or updating contacts.

Read-only mounts:
- `/sendgrid/mail/<messageId>.json` - Mail send records.
- `/sendgrid/contacts/<contactId>.json` - Marketing contact records.

Resources:

| Resource | Schema | Create example | ID pattern | What it does |
|---|---|---|---|---|
| `/sendgrid/mail/<id>.json` | `/sendgrid/mail/.schema.json` | `/sendgrid/mail/.create.example.json` | `^(?!(?:new\|create\|draft\|send)(?:[-_\s].*)?$)[A-Za-z0-9_.:-]+$` | Sends an email through SendGrid Mail Send. |
| `/sendgrid/contacts/<id>.json` | `/sendgrid/contacts/.schema.json` | `/sendgrid/contacts/.create.example.json` | `^(?!(?:new\|create\|draft\|send)(?:[-_\s].*)?$)[A-Za-z0-9_.:-]+$` | Upserts one or more SendGrid marketing contacts. |

## Operations

| To... | Do... |
|---|---|
| Read | `cat <canonical-resource-path>` after listing the resource directory or following an alias when one is available. Use the resource table and ID patterns below to determine whether a resource uses a bare id, an adapter-specific slug/id filename, or an exact sidecar path such as `content.md`. |
| Edit | Write the resource update payload to the canonical resource path. For JSON resources, included mutable fields PATCH; fields marked `readOnly` in `.schema.json` are rejected. |
| Create | Write JSON to any non-canonical filename such as `create request.json`. The adapter creates the record at its canonical resource path and rewrites the draft as `{ "created": "<real-id>", "path": "<canonical-resource-path>", "url": "<provider-url>" }`. |
| Ignore | Editor scratch files named `partial.json`, `.tmp.json`, `.partial.json`, `*.tmp.json`, or `*.partial.json` are ignored and never treated as create drafts. |
| Delete | `rm <canonical-resource-path>` for canonical records. |

## ID Patterns
- `/sendgrid/mail/<id>.json`: `^(?!(?:new|create|draft|send)(?:[-_\s].*)?$)[A-Za-z0-9_.:-]+$`. Filenames that do not match this pattern are treated as create drafts.
- `/sendgrid/contacts/<id>.json`: `^(?!(?:new|create|draft|send)(?:[-_\s].*)?$)[A-Za-z0-9_.:-]+$`. Filenames that do not match this pattern are treated as create drafts.

## Write field contracts

### Send SendGrid mail

Resource: `/sendgrid/mail/<id>.json`
Schema: `/sendgrid/mail/.schema.json`
Create example: `/sendgrid/mail/.create.example.json`
Required fields: `from`, `personalizations`.
Optional fields: `subject`, `template_id`, `categories`, `custom_args`, `mail_settings`, `tracking_settings`, `send_at`, `content`, `reply_to`.

Fields:

- `from` (required, object) - Sender address.
- `from.email` (required, string, email) - Sender email address.
- `from.name` (optional, string) - Sender display name.
- `personalizations` (required, array) - SendGrid personalization entries.
- `subject` (optional, string) - Message subject.
- `template_id` (optional, string) - Dynamic template id.
- `categories` (optional, array) - Message categories.
- `custom_args` (optional, object) - Custom argument map.
- `mail_settings` (optional, object) - SendGrid mail settings.
- `tracking_settings` (optional, object) - SendGrid tracking settings.
- `send_at` (optional, integer) - Unix timestamp for scheduled send.
- `content` (optional, array) - Message content blocks.
- `reply_to` (optional, object) - Reply-To address.
- `reply_to.email` (optional, string, email) - Reply-To email address.
- `reply_to.name` (optional, string) - Reply-To display name.

### Upsert SendGrid contacts

Resource: `/sendgrid/contacts/<id>.json`
Schema: `/sendgrid/contacts/.schema.json`
Create example: `/sendgrid/contacts/.create.example.json`
Required fields: `email`.
Optional fields: `contacts`, `list_ids`, `first_name`, `last_name`, `address_line_1`, `address_line_2`, `city`, `state_province_region`, `postal_code`, `country`, `phone_number`, `whatsapp`, `line`, `facebook`, `unique_name`, `alternate_emails`, `custom_fields`.

Fields:

- `email` (required, string, email) - Contact email address.
- `contacts` (optional, array) - Contacts to upsert.
- `list_ids` (optional, array) - Marketing list ids.
- `first_name` (optional, string) - Contact first name.
- `last_name` (optional, string) - Contact last name.
- `address_line_1` (optional, string) - Contact address line 1.
- `address_line_2` (optional, string) - Contact address line 2.
- `city` (optional, string) - Contact city.
- `state_province_region` (optional, string) - Contact state, province, or region.
- `postal_code` (optional, string) - Contact postal code.
- `country` (optional, string) - Contact country.
- `phone_number` (optional, string) - Contact phone number.
- `whatsapp` (optional, string) - WhatsApp contact value.
- `line` (optional, string) - LINE contact value.
- `facebook` (optional, string) - Facebook contact value.
- `unique_name` (optional, string) - Provider unique contact name.
- `alternate_emails` (optional, array) - Alternate email addresses.
- `custom_fields` (optional, object) - SendGrid custom field map.
