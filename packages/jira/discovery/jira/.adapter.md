# Jira adapter

The Jira adapter exposes issues, comments, projects, and sprints under `/jira`, with writeback routes for creating issues, projects, and issue comments.

Read-only mounts:
- `/jira/issues/<issueIdOrKey>.json` - Issue records.
- `/jira/issues/<issueIdOrKey>/comments/<commentId>.json` - Issue comment records.
- `/jira/projects/<projectIdOrKey>.json` - Project records.
- `/jira/sprints/<sprintId>.json` - Sprint records.

Resources:

| Resource | Schema | Create example | ID pattern | What it does |
|---|---|---|---|---|
| `/jira/issues/{issueIdOrKey}/comments/<id>.json` | `/jira/issues/{issueIdOrKey}/comments/.schema.json` | `/jira/issues/{issueIdOrKey}/comments/.create.example.json` | `^\d+$` | Adds a comment to a Jira issue. |
| `/jira/issues/<id>.json` | `/jira/issues/.schema.json` | `/jira/issues/.create.example.json` | `^(?:[A-Z][A-Z0-9]+-\d+|\d+)$` | Creates a Jira issue. |
| `/jira/projects/<id>.json` | `/jira/projects/.schema.json` | `/jira/projects/.create.example.json` | `^(?:[A-Z][A-Z0-9]+-\d+|\d+)$` | Creates a Jira project. |

## Operations

| To... | Do... |
|---|---|
| Read | `cat <id>.json` |
| Edit | Write a partial JSON object to `<id>.json`. Only included mutable fields PATCH; fields marked `readOnly` in `.schema.json` are rejected. |
| Create | Write JSON to any non-canonical filename such as `create request.json`. The adapter creates the record at `<real-id>.json` and rewrites the draft as `{ "created": "<real-id>", "path": "<resource>/<real-id>.json", "url": "<provider-url>" }`. |
| Delete | `rm <id>.json` for canonical ids. |

## ID Patterns
- `/jira/issues/{issueIdOrKey}/comments/<id>.json`: `^\d+$`. Filenames that do not match this pattern are treated as create drafts.
- `/jira/issues/<id>.json`: `^(?:[A-Z][A-Z0-9]+-\d+|\d+)$`. Filenames that do not match this pattern are treated as create drafts.
- `/jira/projects/<id>.json`: `^(?:[A-Z][A-Z0-9]+-\d+|\d+)$`. Filenames that do not match this pattern are treated as create drafts.

## Write field contracts

### Create Jira issue comment

Resource: `/jira/issues/{issueIdOrKey}/comments/<id>.json`
Schema: `/jira/issues/{issueIdOrKey}/comments/.schema.json`
Create example: `/jira/issues/{issueIdOrKey}/comments/.create.example.json`
Required fields: `body`.
Optional fields: none.

Fields:

- `body` (required, one of object, string) - Jira rich-text document body, or plain string body.

### Create Jira issue

Resource: `/jira/issues/<id>.json`
Schema: `/jira/issues/.schema.json`
Create example: `/jira/issues/.create.example.json`
Required fields: `fields`.
Optional fields: none.

Fields:

- `fields` (required, object) - Jira issue fields.
- `fields.project` (required, object) - Project object. Usually includes `key` or `id`.
- `fields.summary` (required, string) - Issue summary.
- `fields.issuetype` (required, object) - Issue type object. Usually includes `name` or `id`.
- `fields.description` (optional, object) - Jira rich-text description document.
- `fields.priority` (optional, object) - Priority object. Usually includes `name` or `id`.
- `fields.labels` (optional, array) - Issue labels.

### Create Jira project

Resource: `/jira/projects/<id>.json`
Schema: `/jira/projects/.schema.json`
Create example: `/jira/projects/.create.example.json`
Required fields: `key`, `name`, `projectTypeKey`, `leadAccountId`.
Optional fields: `description`, `url`, `assigneeType`.

Fields:

- `key` (required, string) - Project key.
- `name` (required, string) - Project name.
- `projectTypeKey` (required, string) - Project type key such as software, business, or service_desk.
- `leadAccountId` (required, string) - Jira account id for the project lead.
- `description` (optional, string) - Project description.
- `url` (optional, string, uri) - Project URL.
- `assigneeType` (optional, string) - Default assignee type.

## Create Examples
Read the resource `.schema.json` first, then use the sibling `.create.example.json` as a minimal create document. The example intentionally omits read-only fields.
