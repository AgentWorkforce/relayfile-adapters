# OneDrive Storage Bridge Adapter

OneDrive maps Microsoft Graph subscriptions plus drive delta into Relayfile StorageBridgeEvent envelopes under /onedrive/{accountId}/{path}. The adapter validates configuration, normalizes push notifications, publishes storage events, fetches content through injected provider clients, and resolves file-native writeback into provider-shaped operations.

## Read-only Mounts

Read-only synchronized content is mounted at `/onedrive/{accountId}/{path}`. Deleted provider objects emit deletion events with null fetch content.

## Writable Resources

The resource `/onedrive/{accountId}/items` ships `/onedrive/{accountId}/items/.schema.json` and `/onedrive/{accountId}/items/.create.example.json`.

The resource `/onedrive/subscriptions` ships `/onedrive/subscriptions/.schema.json` and `/onedrive/subscriptions/.create.example.json`.

The static discovery resource `/onedrive/items/.schema.json` ships `/onedrive/items/.create.example.json`.

## Operations

Create writes use any non-canonical filename in the resource directory. Canonical filenames patch mutable fields only. Removing a canonical file resolves to provider delete operations. Read-only fields such as `id`, `createdAt`, `updatedAt`, `url`, `_webhook`, and `_connection` are rejected before provider calls.

The object resource resolves to `/v1.0/me/drive/items/{itemId}`. The lifecycle resource resolves to `/v1.0/subscriptions`.

## Push and Auth Validation

Graph validationToken challenges are returned before event normalization; local signatures use x-relayfile-signature HMAC.

## Nango template

Nango anchor: `one-drive`; sync `onedrive-files`; scopes `Files.ReadWrite.All` and `offline_access`.

## Nango Fallback

OneDrive declares `onedrive-files` for scheduled fallback records using provider config key `microsoft`.

## ID Patterns

`/onedrive/{accountId}/items`: `^[A-Za-z0-9!._-]+$`.

`/onedrive/subscriptions`: `^[A-Za-z0-9_-]+$`.
